# 第11节：课前检查与材料

## 课前15分钟检查清单

课前15分钟到教室，按以下顺序逐项检查：

### 设备检查（5分钟）

- [ ] 投影仪开机，连接讲师电脑，画面正常
- [ ] 讲师电脑终端打开，字体大小调到后排能看清（建议18-20号字）
- [ ] 每台学员电脑开机，WSL 终端可正常打开
- [ ] 每台学员电脑 Claude Code 可用：在终端输入 `claude --version`，确认有输出
- [ ] 每台学员电脑浏览器可用：双击打开浏览器，确认能正常显示页面

### 网络检查（3分钟）

- [ ] 教室 Wi-Fi 正常连接
- [ ] 在讲师电脑上测试 Claude Code 能正常响应：`claude "请说你好"`
- [ ] 在至少2台学员电脑上测试 Claude Code 能正常响应

### 演示环境准备（5分钟）

- [ ] 在**讲师电脑**上创建工作目录：
  ```bash
  mkdir -p ~/lesson11 && cd ~/lesson11
  ```
- [ ] 在**讲师电脑**上测试浏览器打开本地 HTML 文件的方式：
  ```bash
  echo "<h1>测试</h1>" > test.html
  explorer.exe test.html
  ```
  确认浏览器能正常打开并显示"测试"二字
- [ ] 如果 `explorer.exe` 方式不行，测试备用打开方式：
  - 在 Windows 文件管理器中导航到 WSL 路径 `\\wsl$\Ubuntu\home\用户名\lesson11\`
  - 双击 HTML 文件打开
  - 记下可用的打开方式，上课时统一教给学员
- [ ] 在**每台学员电脑**上创建工作目录：
  ```bash
  mkdir -p ~/lesson11
  ```
- [ ] 删除测试文件：`rm -f ~/lesson11/test.html`

### 材料确认（2分钟）

- [ ] 网页应用方向参考已准备好（见本文档"教学素材"部分），可投影展示

```html
            border-radius: 8px;
            margin-bottom: 8px;
            box-shadow: 0 1px 4px rgba(0,0,0,0.05);
        }
        .record .purpose { color: #333; }
        .record .amount { color: #e74c3c; font-weight: bold; }
    </style>
</head>
<body>
    <h1>个人记账器</h1>
    <div class="total">总花费：<span id="total">0</span> 元</div>
    <div class="input-group">
        <input type="number" id="amount" placeholder="金额">
        <input type="text" id="purpose" placeholder="用途">
        <button onclick="addRecord()">记一笔</button>
    </div>
    <div id="records"></div>
    <script>
        let records = [];
        function addRecord() {
            let amount = parseFloat(document.getElementById('amount').value);
            let purpose = document.getElementById('purpose').value;
            if (!amount || !purpose) { alert('请输入金额和用途'); return; }
            records.push({ amount, purpose });
            document.getElementById('amount').value = '';
            document.getElementById('purpose').value = '';
            renderRecords();
        }
        function renderRecords() {
            let total = records.reduce((sum, r) => sum + r.amount, 0);
            document.getElementById('total').textContent = total.toFixed(2);
            let html = records.map(r =>
                `<div class="record"><span class="purpose">${r.purpose}</span><span class="amount">-${r.amount.toFixed(2)} 元</span></div>`
            ).reverse().join('');
            document.getElementById('records').innerHTML = html;
        }
    </script>
</body>
</html>
```

## 备用方案

### 场景A：浏览器无法打开 WSL 中的 HTML 文件

**现象**：`explorer.exe` 命令执行后浏览器没反应，或者报错。

**处理步骤：**

1. 尝试用完整路径：
   ```bash
   explorer.exe $(wslpath -w ~/lesson11/calculator.html)
   ```
2. 如果还是不行，让学员通过 Windows 文件管理器手动找到文件：
   - 打开文件管理器，地址栏输入 `\\wsl$\Ubuntu\home\`
   - 找到用户名文件夹 → lesson11 → 双击 .html 文件
3. 如果以上都不行：
   - 让 Claude Code 把 HTML 文件保存到 Windows 桌面：
   ```
   把文件保存到 /mnt/c/Users/用户名/Desktop/calculator.html
   ```
   - 然后在桌面直接双击打开

### 场景B：Claude Code 完全不可用

1. 使用上方备用代码，直接复制到学员电脑
2. 让学员在浏览器中打开备用的计算器和记账器
3. 教学内容调整为：
   - 讲师讲解 HTML/CSS/JS 的角色分工（用备用代码做示例）
   - 学员练习：手动修改 HTML 中的文字内容、CSS 中的颜色值
   - 展示环节不变
4. 跟学员说："今天网络有问题，我们用现成的代码来学习网页的结构。下次课 AI 恢复了我们再做自己的项目。"

### 场景C：个别学员电脑有问题

- 让该学员和旁边的同学共用一台电脑
- 尹伟优先协助排查该电脑问题
- 两人共用时建议：一人先做，另一人观摩帮忙出主意，中间换人

### 场景D：学员不知道做什么，选择困难

- 直接指定一个方向："你先做这个倒计时器，最简单，5分钟就能看到效果"
- 把提示词模板投影出来，让学员直接复制粘贴，只改工具名称和功能
