# 第11节：实践 -- 网页应用开发

## 课时信息

| 项目 | 内容 |
|:---|:---|
| 所属阶段 | 实践课（第3节/共5节） |
| 主讲 | 沈林伟 |
| 辅助 | 尹伟（巡场、辅助答疑） |
| 时长 | 45 分钟 |
| 前置要求 | 学员已完成第9-10节实践课，能用 Claude Code 独立完成脚本开发 |

## 教学目标

学完本节课，学员能够：

1. 用 Claude Code 生成一个可在浏览器中打开的单页网页应用
2. 理解网页的三个组成部分（HTML=内容、CSS=样式、JS=交互）
3. 修改网页的内容、颜色、布局
4. 理解"网页应用"和"命令行脚本"的区别——有界面、能点击、能分享

## 核心理念（上课前讲师务必读）

> **本节课是学员第一次做"能看到界面"的东西。**
>
> 前两节实践课做的都是命令行脚本——运行结果是一行行文字。这节课开始，学员将看到自己用 AI 做出来的东西可以在浏览器里打开、能点击、有颜色。
>
> 这种"我做了一个能看到的东西"的成就感非常强烈，要充分利用这个正向动力。不要在技术概念上过度深入，重点是让每个人都做出自己的网页并展示。
>
> HTML/CSS/JS 只需要最表层的理解："改文字改HTML、改颜色改CSS、改功能改JS"。够用就好。

---

## 分层要求

| 等级 | 完成标准 |
|:---:|:---|
| L1 能用 | 用 Claude Code 生成网页，能在浏览器中打开，基本功能正常工作 |
| L2 能调 | L1 + 修改页面内容、颜色、布局；能说出 HTML/CSS/JS 各自负责什么 |
| L3 能交付 | L2 + 添加交互功能（如数据保存到本地存储、输入校验）；页面美观、完整，可以发给别人使用 |

## 教学流程

### 第一部分：学员回顾（5分钟）[0:00-0:05]

**讲师操作：**

> 讲师说："我们已经做了两节实践课了——自动化脚本和数据处理工具。谁来说说，到目前为止用 Claude Code 做项目的流程是什么？"

- 请1-2位学员简要回顾
- 如果学员说不出来，讲师引导："我们做一个东西，一般分哪几步？先做什么，再做什么？"

**预期回答要点：**
- 先用中文写清楚需求
- 用 Claude Code 生成代码
- 运行看结果
- 不满意就让 Claude Code 修改

**讲师总结（30秒）：**

> "没错，流程就是这样：想清楚→描述需求→生成代码→运行→改。今天流程一模一样，但做出来的东西不一样了——今天我们不做命令行脚本了，我们要做**网页**。你做出来的东西可以在浏览器里打开，有界面、有按钮、能点击。"

---

### 第二部分：讲师演示 -- 3分钟做一个计算器网页（3分钟）[0:05-0:08]

> **设计意图**：用一个快速演示让学员看到"从零到一个能用的网页"只需要3分钟。建立信心，同时展示完整流程。

**讲师台词：**

> "大家看我的屏幕。我现在用 Claude Code 做一个计算器网页。计时开始。"

**讲师操作（投影演示）：**

```bash
cd ~/lesson11
claude
```

**在 Claude Code 中输入：**

```
做一个计算器网页应用。要求：
1. 页面有数字按钮0-9，加减乘除按钮，等号和清除按钮
2. 点击按钮输入算式，点等号显示结果
3. 界面美观，按钮大一点方便点击
4. 保存为 calculator.html 一个文件
```

**等待 Claude Code 生成（大约1-2分钟）：**

> 讲师在等待时可以说："大家注意看，我只需要用中文描述我想要什么，Claude Code 就会帮我写出 HTML、CSS 和 JavaScript 代码。"

**生成后，在浏览器中打开：**

在 WSL 中运行：
```bash
explorer.exe calculator.html
```

或者告诉学员如何在 Windows 文件管理器中找到文件双击打开。

**讲师台词（指着浏览器中的计算器页面）：**

> "看！3分钟不到，一个能用的计算器就出来了。你们可以点按钮试试——这就是一个完整的网页应用。"
>
> "跟之前做的脚本有什么不同？脚本运行完就是一堆文字。这个有界面、有按钮、能交互。而且你可以把这个文件发给任何人，他们在浏览器里打开就能用。"

**快速讲解 HTML/CSS/JS（30秒，点到即止）：**

> "这个网页只有一个文件，但里面包含三样东西：
> - **HTML** 负责内容——按钮上写什么字、页面有哪些元素
> - **CSS** 负责样式——按钮多大、什么颜色、怎么排列
> - **JavaScript** 负责交互——点按钮之后发生什么
>
> 你不需要记住这些术语。只需要知道：**改文字找 HTML，改颜色找 CSS，改功能找 JavaScript。** 而且你可以直接用中文告诉 Claude Code 要改什么，它会帮你改对地方。"

---

### 第三部分：选择方向 + 写需求（4分钟）[0:08-0:12]

**讲师台词：**

> "现在轮到你们了。每个人要做一个自己的网页应用。你可以自己想做什么，也可以从下面这几个方向里选一个。"

**投影展示可选方向：**

```
网页应用方向参考（选一个，或者自己想）

1. 个人记账器
   - 输入金额和用途，点按钮记录
   - 显示所有记录和总花费

2. 单位换算工具
   - 输入一个数字，选择换算类型（斤↔公斤、人民币↔美元等）
   - 自动显示换算结果

3. 倒计时器
   - 输入一个目标日期（比如生日、发工资的日子）
   - 显示距离那一天还有多少天

4. 知识问答小游戏
   - 显示一道题目，给出几个选项
   - 点选项后告诉你对不对，记录得分
```

**讲师台词：**

> "现在每个人做两件事：
> 1. 选一个方向（可以是上面的，也可以自己想的）
> 2. 用中文写出你要什么——不需要写很多，2-3句话就够。比如'我要做一个记账的网页，能输入金额和用途，能看到所有记录和总花费'。
>
> 写好需求之后就可以开始用 Claude Code 做了。4分钟之内选好、写好、开始做。"

**尹伟操作：** 在学员之间走动，帮没有想法的学员选方向、帮不知道怎么写需求的学员组织语言。

---

### 第四部分：独立开发（23分钟）[0:12-0:35]

> **设计意图**：这是本节课的主体时间。每个学员用 Claude Code 独立开发自己的网页应用。讲师和助教巡场指导。

**讲师台词（开发前提醒）：**

> "几个提醒：
> 1. 先创建工作目录：`mkdir -p ~/lesson11 && cd ~/lesson11`
> 2. 用 Claude Code 时，告诉它'保存为一个 HTML 文件'。一个文件最方便，打开就能用
> 3. 生成完之后，用浏览器打开看效果。不满意就继续跟 Claude Code 说要改什么
> 4. 别追求完美——先做出能用的版本，再慢慢改
>
> 好，开始！"

#### 巡场指导要点

**讲师（沈林伟）巡场重点：**

| 观察到的情况 | 指导方式 |
|:---|:---|
| 学员还没开始写需求 | "你想做什么？说出来就行，我帮你理一下" |
| 需求写得太模糊（"做一个好看的网页"） | "好看是什么样的？有什么功能？谁来用？比如说'我要做一个记账的，能输入金额'" |
| Claude Code 生成了代码但学员不知道怎么打开 | 演示：`explorer.exe 文件名.html`，或者在 Windows 文件管理器中找到文件双击 |
| 网页打开了但有 bug | "你看到什么问题了？把问题描述给 Claude Code，比如'点按钮没反应'或者'数字显示不对'" |
| 学员只做到 L1 就停了 | 引导升级："这个基本功能有了。你觉得颜色好不好看？想不想换个颜色？跟 Claude Code 说'把背景色改成浅蓝色'" |
| 学员做得很快，已经到 L2 | 引导进阶："试试加一个功能——比如让数据保存下来，关掉浏览器再打开还在。跟 Claude Code 说'用 localStorage 保存数据'" |

**助教（尹伟）巡场重点：**

| 优先级 | 任务 |
|:---|:---|
| 第一优先 | 帮卡在环境问题的学员（浏览器打不开文件、Claude Code 不响应等） |
| 第二优先 | 帮进度慢的学员——如果有人5分钟了还没生成出代码，主动过去协助 |
| 第三优先 | 确认每个学员都在写自己选的方向，没有抄别人的（鼓励独立思考） |

#### 分层进度检查点

在开发过程中（大约20分钟时，即 0:32 左右），讲师快速巡一圈，心里记录每个人的进度：

| 进度 | 对应 | 预期人数 |
|:---|:---|:---|
| 网页已经在浏览器里打开，基本功能能用 | L1 已达成 | 大部分学员 |
| 正在修改样式或添加功能 | 向 L2 前进 | 部分学员 |
| 功能完善，界面美观，有额外交互 | 向 L3 前进 | 个别学员 |
| 还在生成第一版 / 卡住了 | 需要帮助 | 个别学员 |

---

### 第五部分：展示环节 -- Show & Tell（10分钟）[0:35-0:45]

> **设计意图**：让每个人都展示自己的作品，互相观摩。这是本节课成就感的高潮环节，也是同伴学习的好机会。

**讲师台词：**

> "好，时间到！不管做到什么程度都没关系——能打开就是成果。现在每个人在自己的浏览器里把网页打开，我们互相看看大家做了什么。"

#### 展示流程

**第一轮：走动浏览（5分钟）**

> "大家站起来，走一圈，看看每个人做了什么。看到有意思的可以问问他是怎么做的。"

**操作：**
1. 确保每台电脑屏幕上都打开了学员的网页作品
2. 学员离开座位，绕教室走一圈，看每个人的屏幕
3. 讲师也跟着走，对每个作品简短点评一句（"这个配色不错"、"这个功能挺实用的"）

**第二轮：亮点分享（5分钟）**

> "好，回到座位。我刚才看了一圈，有几个作品想请作者给大家介绍一下。"

**挑2-3个有代表性的作品，请学员简短介绍（每人1分钟）：**

- 选一个功能做得扎实的（体现 L1/L2）
- 选一个界面特别好看的（体现 CSS 修改能力）
- 选一个有创意方向的（鼓励独立思考）

**讲师引导学员介绍时说：**

> "你做的是什么？简单说说。"
> "这个功能你是怎么跟 Claude Code 描述的？"

**讲师总结（30秒）：**

> "今天每个人都做了自己的第一个网页应用。跟之前的脚本不一样，这个东西有界面、能交互、可以发给别人用。
>
> 记住三个关键词：HTML 管内容、CSS 管样式、JS 管交互。但最重要的不是记住术语——是你们已经证明了自己能用 AI 做出有界面的应用。
>
> 下节课我们要做自选项目——你自己决定做什么。今天的网页应用也可以继续完善，变成你的最终项目。"

## 常见问题与应对（FAQ）

| 学员问题 | 讲师回应 |
|:---|:---|
| "浏览器打开是空白页 / 乱码" | "这可能是文件没保存完整。回到 Claude Code 让它重新生成一下。或者检查文件名是不是 .html 结尾的。" |
| "我点按钮没反应" | "这说明 JavaScript 部分可能有问题。你跟 Claude Code 说'点击按钮没有反应，请帮我检查 JavaScript 部分'。" |
| "怎么在浏览器里打开本地文件？" | "在 WSL 里输入 `explorer.exe 文件名.html`。或者打开 Windows 文件管理器，找到 WSL 里的文件夹，双击 HTML 文件。" |
| "HTML/CSS/JS 分别是什么？" | "简单说：HTML 写内容（页面上有什么），CSS 写样式（长什么样），JavaScript 写功能（点了之后做什么）。你不需要自己写，让 AI 帮你写，你只需要知道改哪里找谁。" |
| "能不能做好几个页面？" | "今天先做好一个页面。一个文件最简单——打开就能用。后面课程可以学更复杂的。" |
| "我的网页能放到网上让别人看吗？" | "可以的，但那是后面的内容。今天先把文件做好。这个 HTML 文件你可以用微信发给朋友，他们保存下来用浏览器打开就能用。" |
| "我做的和别人的方向一样怎么办？" | "没关系，同样的方向每个人做出来肯定不一样。就像两个人都做记账，你关注生活费他关注工作花销，需求不同，结果就不同。" |
| "能不能做一个手机能用的网页？" | "可以！跟 Claude Code 说'请让这个网页在手机上也能正常显示，用响应式布局'。但这是加分项，先把基本功能做好。" |

## 时间应急方案

| 情况 | 方案 |
|:---|:---|
| 演示环节超时（超过8分钟） | 压缩选方向的时间——直接让学员从4个方向里选一个，跳过自由发挥。确保开发时间不少于20分钟 |
| 大部分学员在12分钟时还没开始开发 | 讲师投影一个通用的提示词模板，让学员直接填空使用（见下方"提示词模板"） |
| Claude Code 响应慢 | 让学员两人共用一个 Claude Code 会话。一个人生成代码时，另一个人先手写需求 |
| 展示环节时间不够（开发超时到40分钟） | 砍掉走动浏览环节，直接让2个学员屏幕投影展示，每人1分钟 |
| 学员进度快，开发提前完成 | 提前进入展示环节，多让几个人分享。或让完成的学员帮还没完成的同学 |

**提示词模板（备用）：**

```
用 Claude Code 时可以直接说：

"做一个 [工具名称] 的网页应用。要求：
1. [第一个功能]
2. [第二个功能]
3. 界面简洁好看
4. 保存为一个 HTML 文件"

例子：
"做一个倒计时器的网页应用。要求：
1. 可以输入一个目标日期
2. 显示距离那个日期还有多少天、多少小时
3. 界面简洁好看
4. 保存为一个 HTML 文件"
```

```

  JS    = 交互（点击之后做什么）

改文字 → 找 HTML
改颜色 → 找 CSS
改功能 → 找 JS
（或者直接用中文告诉 Claude Code 要改什么！）

操作流程：
1. mkdir -p ~/lesson11 && cd ~/lesson11
2. claude
3. 用中文描述需求（"保存为一个 HTML 文件"）

5. 不满意？继续跟 Claude Code 说要改什么
```
