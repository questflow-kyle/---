# 第12节：课前检查与材料

## 课前15分钟检查清单

课前15分钟到教室，按以下顺序逐项检查：

### 设备检查（5分钟）

- [ ] 投影仪开机，连接讲师电脑，画面正常
- [ ] 讲师电脑终端打开，字体大小调到后排能看清（建议18-20号字）
- [ ] 每台学员电脑开机，WSL 终端可正常打开
- [ ] 每台学员电脑 Python 可用：在终端输入 `python3 --version`，确认有输出
- [ ] 每台学员电脑 Claude Code 可用：在终端输入 `claude --version`，确认有输出
- [ ] 每台学员电脑浏览器可用（部分学员可能做网页项目）

### 网络检查（3分钟）

- [ ] 教室 Wi-Fi 正常连接
- [ ] 在讲师电脑上测试 Claude Code 能正常响应：`claude "请说你好"`
- [ ] 在至少2台学员电脑上测试 Claude Code 能正常响应
- [ ] 测试网络速度：Claude Code 响应时间在30秒以内为正常

### 材料准备（5分钟）

- [ ] 项目方向参考清单已准备好，可投影展示（见本文档"教学素材"部分）
- [ ] 需求描述模板已打印，**每人一张**（见单独文件：需求描述模板.md）
- [ ] 结对互审流程已准备好，可投影展示（见本文档"教学素材"部分）
- [ ] 在**每台学员电脑**上创建项目工作目录：
  ```bash
  mkdir -p ~/project
  ```

### 讲师沟通确认（2分钟）

- [ ] 三位讲师（沈林伟、梅杭俊、尹伟）都到场
- [ ] 确认巡场分工：
  - 沈林伟：技术指导
  - 梅杭俊：需求梳理
  - 尹伟：环境保障
- [ ] 确认结对互审的计时方式（手机闹钟/看手表）

## 备用方案

### 场景A：Claude Code 完全不可用

**处理步骤：**

1. 不要慌，本节课的核心是"定义问题和描述需求"
2. 让学员先完成需求描述模板的填写（这不需要电脑）
3. 需求描述完成后，尝试以下替代方案：
   - 如果网页版 Claude (claude.ai) 可用：让学员在网页上与 Claude 对话生成代码，手动复制到文件
   - 如果完全没有 AI 可用：讲师帮学员用预先准备的模板代码修改出他们想要的功能
4. 结对互审环节正常进行——可以互审需求描述和设计思路，不一定非要看运行的代码
5. 跟学员说："今天我们重点练习的是发现问题和描述需求。代码部分下节课补上。"

### 场景B：个别学员全程没想法

**处理步骤：**

1. 梅杭俊在前5分钟重点关注这类学员
2. 用引导式提问帮学员找方向：
   - "你每天起床到睡觉，有什么事情是重复做的？"
   - "你工作的时候有没有什么事情每次都要手算？"
   - "你手机上最常用的3个App是什么？有没有哪个功能你觉得可以更好？"
3. 如果5分钟后还是没想法，直接指定："你做这个工资计算器吧，最贴近你的工作。"

### 场景C：学员电脑有问题

- 让该学员和旁边的同学共用一台电脑
- 尹伟优先排查
- 两人共用时可以做同一个项目的不同功能（一人做界面一人做逻辑），互审时分开展示各自的部分

### 场景D：结对互审时一组完全沉默

**讲师走过去引导：**

> "来，你先给他看看你做了什么。打开它，让他看。"
> （等展示后）"你觉得他这个工具怎么样？如果你来用，第一步你会做什么？"
> （如果还是沉默）"你觉得有什么功能可以加上去？或者有什么地方用起来不太方便？"

### 场景E：学员人数为单数，无法两两配对

- 安排一组三人：每人2分钟展示，每轮另外两人都可以提问和建议
- 或者让落单的学员和一位讲师配对互审
