# 答辩提问库

> 讲师在答辩过程中参考使用。根据学员展示的水平，从对应类别中选择合适的问题。
>
> **使用原则：**
> - 每位学员问1-2个问题即可，不要问太多
> - 根据学员水平选择难度：展示紧张的问基础题，展示流畅的可以追问
> - 目的是帮学员"说出来"，不是"考倒他"
> - 如果学员答不上来，换一个更简单的问题，或者给提示引导

---

## 一、需求理解（考察：为什么做这个）

> **适合所有学员**。这是最容易回答的问题类型，适合作为开场或帮紧张的学员破冰。

| 编号 | 问题 | 追问方向 |
|:---:|:---|:---|
| 需1 | "你怎么想到做这个项目的？" | 是自己想的还是参考了清单？为什么选这个方向？ |
| 需2 | "这个程序主要给谁用的？" | 是给自己用、给同事用、还是给家人用？ |
| 需3 | "你觉得这个工具解决了什么问题？" | 没有这个工具之前你怎么做这件事？ |
| 需4 | "如果重新做一遍，你会改什么？" | 需求会变吗？还是做法会变？ |
| 需5 | "你一开始想做的和最后做出来的一样吗？" | 哪里做了妥协？为什么？ |

---

## 二、代码理解（考察：是否读懂了自己的代码）

> **主要用于判断L1/L2分界**。L1只需要说出"做什么"，L2需要说出"怎么做的"。

| 编号 | 问题 | 追问方向 |
|:---:|:---|:---|
| 码1 | "你能打开代码，指给我看哪一段是做[核心功能]的吗？" | 看学员能否在代码中定位功能对应的位置 |
| 码2 | "这段代码是做什么的？用你自己的话说。" | 指向代码中一段关键逻辑，看学员能否解释 |
| 码3 | "如果我想把[某个参数]从X改成Y，应该改哪里？" | 比如把及格线从60改成70、把颜色从蓝改成红 |
| 码4 | "这里有一个 if 判断，它在判断什么条件？" | 看学员是否理解条件逻辑 |
| 码5 | "这里有一个循环，它在对什么东西做重复操作？" | 看学员是否理解循环逻辑 |
| 码6 | "你的程序一共有几个文件？各自做什么？" | 看学员是否了解项目结构（L3方向） |

**提问技巧：**
- 不要问太抽象的（"解释一下这个函数"），要具体（"这行代码在做什么"）
- 如果学员说不出来，可以给提示："这行有个 `if`，`if` 是做什么用的？还记得吗？"

---

## 三、调试能力（考察：遇到问题怎么解决的）

> **这是L2的核心考察点**。区分"让AI修"和"自己先定位再让AI修"。

| 编号 | 问题 | 追问方向 |
|:---:|:---|:---|
| 调1 | "开发过程中遇到的最大的坑是什么？" | 让学员描述具体问题，看是否记得问题细节 |
| 调2 | "你怎么发现问题在哪的？" | 是看了报错信息？还是一行一行排查的？还是直接让AI修？ |
| 调3 | "报错信息说了什么？你当时看懂了吗？" | 看学员是否有"读报错信息"的习惯 |
| 调4 | "你修了几次才修好？" | 了解调试过程是否曲折，学员是否有耐心 |
| 调5 | "如果现在程序突然报一个新的错，你会怎么做？" | 看学员是否掌握了调试四步法（读报错→缩小范围→定位→修复） |

**提问技巧：**
- 如果学员说"AI帮我修的"，追问"你是怎么跟AI说的？是说'帮我修'，还是先告诉它报了什么错？"
- 这个追问本身就是在考察L1和L2的分界线

---

## 四、工具使用（考察：如何与AI协作）

> **适合所有学员**。了解学员使用 Claude Code 的方式和熟练度。

| 编号 | 问题 | 追问方向 |
|:---:|:---|:---|
| 工1 | "你是怎么跟 Claude Code 描述这个需求的？" | 是一句话说完还是分步描述？ |
| 工2 | "Claude Code 第一次生成的代码就能用吗？如果不能，你怎么改的？" | 看学员是否经历了迭代过程 |
| 工3 | "有没有 Claude Code 理解错你意思的时候？你怎么纠正它的？" | 看学员是否会调整提示词 |
| 工4 | "除了 Claude Code，你还用了什么方式解决问题？" | 有没有搜索？有没有看文档？有没有问同学？ |
| 工5 | "如果 Claude Code 不能用了，你觉得你还能不能继续做这个项目？" | 了解学员对工具的依赖程度和独立能力 |

**提问技巧：**
- 工5这个问题比较尖锐，只适合对L2/L3水平的学员提问
- 对L1学员，问工1和工4就够了

---

## 五、拓展思考（考察：能否举一反三）

> **主要用于判断L2/L3分界**。看学员是否有进一步发展的潜力。

| 编号 | 问题 | 追问方向 |
|:---:|:---|:---|
| 拓1 | "如果要给这个项目加一个新功能，你会加什么？" | 看想法是否合理、是否能说出大致实现思路 |
| 拓2 | "你觉得这个项目还有什么不足？" | 看学员是否有自我评估能力 |
| 拓3 | "如果要给别人用这个工具，你觉得还需要做什么？" | 考察"交付意识"——使用说明、错误处理、界面友好度 |
| 拓4 | "如果数据量变大（比如从10条变成10000条），你的程序还能用吗？" | 考察对程序局限性的认知（只适合L3方向的学员） |
| 拓5 | "这门课里学到的东西，你觉得以后工作中哪些能用上？" | 开放性问题，了解学员对课程价值的感受 |

**提问技巧：**
- 拓1是最好用的问题，几乎对所有学员都适用
- 拓4和拓5只适合表现突出的学员，不要对紧张的学员提问

---

## 按学员水平的推荐提问组合

### 学员展示水平偏L1（项目简单、表达紧张）

推荐提问：
1. **需1** "你怎么想到做这个的？"（破冰，最容易回答）
2. **工1** "你是怎么跟 Claude Code 说的？"（了解工具使用方式）

### 学员展示水平偏L2（项目能跑、能基本解释）

推荐提问：
1. **码2** "这段代码是做什么的？"（确认代码理解能力）
2. **调2** "你怎么发现问题在哪的？"（确认调试能力）

### 学员展示水平偏L3（项目完整、表达清楚）

推荐提问：
1. **码6** "你的项目一共有几个文件？各自做什么？"（确认结构理解）
2. **拓1** "如果要加新功能，你会加什么？怎么做？"（确认拓展能力）

---

## 救场问题（学员完全卡住时使用）

如果学员紧张到说不出话，用以下问题引导：

1. "你最开始是怎么跟 Claude Code 说的？还记得你打了什么字吗？"
2. "这个程序你运行过多少次？第一次就成功了吗？"
3. "你觉得做这个项目最难的部分是什么？"
4. "你有没有让旁边的同学帮过你？他帮了什么？"

> 这些问题都是开放性的、没有标准答案的，学员只要开口说就行。目的是**帮学员说出来**，而不是考他。
