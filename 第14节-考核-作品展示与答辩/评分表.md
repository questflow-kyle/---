# 答辩评分表

> 每位讲师一份。答辩过程中逐人记录，合议时对照使用。

---

## 评分标准速览

| 等级 | 核心判断 |
|:---|:---|
| **L1 能用** | 项目能运行，能说出做什么，用了AI和搜索 |
| **L2 能调** | + 能解释代码主要逻辑，做过有方向的调整 |
| **L3 能交付** | + 功能完整，有使用说明，能讲清结构，尝试了部署 |

**L1 检查项：**
- [ ] 项目能运行，有基本功能
- [ ] 能说出项目做什么
- [ ] 开发中用到了AI和搜索

**L2 额外检查项：**
- [ ] 能解释代码的主要逻辑（"这段是做什么的"）
- [ ] 做过有方向的调整（不只是"AI帮我做的"）
- [ ] 遇到问题时能缩小范围定位（不是只说"报错了"）

**L3 额外检查项：**
- [ ] 功能完整，不只是"能跑"而是"能用"
- [ ] 有使用说明（README或口头能说清运行步骤）
- [ ] 能讲清项目结构（哪些文件、各自做什么）
- [ ] 尝试了部署（本地服务器/GitHub Pages/打包）
