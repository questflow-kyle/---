# 第14节：课前检查与材料

## 课前15分钟检查清单

> 全体讲师（沈林伟、梅杭俊、尹伟）在上课前15分钟到达教室，按以下顺序逐项检查。

### 设备检查（5分钟）

- [ ] 投影仪开机，连接讲师电脑，画面正常
- [ ] 投影字体大小后排能看清
- [ ] 每台学员电脑开机，确认学员项目文件还在
  - 逐台打开终端，`ls` 看一下项目目录是否存在
  - 如果有学员项目文件丢失 → 看是否有备份，或让学员口述展示
- [ ] 如果需要学员投屏展示：测试投屏线/无线投屏是否可用
  - 备选方案：不投屏，讲师和其他学员围过来看学员电脑

### 网络检查（2分钟）

- [ ] 教室 Wi-Fi 正常连接
- [ ] 确认 Claude Code 可用（以防学员答辩时需要临时调试）

### 教学材料确认（3分钟）

- [ ] **评分表**：已打印，**每位讲师一份**（见"评分表.md"）
  - 确认评分表上的学员名单与实际到场人数一致
  - 如果有学员缺席，在评分表上标记
- [ ] **答辩提问库**：已打印，**每位讲师一份**（见"答辩提问库.md"）
- [ ] **笔**：每位讲师一支笔，用于评分表记录

### 环境布置（3分钟）

- [ ] **座位安排**：学员坐成半圆形或U形，方便互相看到展示
- [ ] **讲师位置**：三位讲师坐在一侧，面对学员，能看到学员电脑屏幕
- [ ] **展示位**：如果用投屏，确认投屏位置和投影画面。如果不投屏，确认讲师能走到学员旁边看屏幕
- [ ] **时间提醒工具**：手机计时器或手表，用于控制每人答辩时间

### 讲师分工确认（2分钟）

三位讲师快速对齐分工：

| 角色 | 职责 |
|:---|:---|
| 沈林伟 | 主持流程、控制时间、开场和收尾台词、宣布结果 |
| 梅杭俊 | 主要负责记录评分、观察学员非语言表现（是否紧张、是否真的理解） |
| 尹伟 | 主要负责技术类提问、协助学员电脑操作（如程序突然跑不了） |

### 最终确认

## 备用方案

### 场景：学员项目文件丢失

**处理步骤：**
1. 检查是否有其他备份目录（如 ~/learning、~/project 等）
2. 检查 Claude Code 历史记录中是否有项目相关的操作记录
3. 如果确实找不到文件：
   - 让学员口述自己的项目是什么、做了什么
   - 讲师可以当场让学员用 Claude Code 快速重建核心部分（如果时间允许）
   - 评分以口述内容和过程理解为主

### 场景：学员缺席

- 如果学员事先请假：安排补考时间，单独评估
- 如果学员未通知缺席：课后联系。可以提交录屏或远程展示
- 把节省的时间分配给其他学员或课程总结

### 场景：答辩时间严重超时

- 如果前面几位学员用时过长（每人超过5分钟）：
  1. 立即调整：后续学员展示限2分钟，提问限1个
  2. 合议环节缩短到3分钟（讲师快速对齐即可）
  3. 课程总结缩短到1分钟，后续建议改为课后发文字版

### 场景：投影仪故障

- 答辩环节不依赖投影——学员在自己电脑上展示，讲师走过去看
- 开场的流程说明可以口述
- 后续建议部分可以课后发文字版给学员

### 场景：某学员答辩时情绪波动（过于紧张/哭泣等）

1. 不要催促，给学员时间平复情绪
2. 可以说"没关系，深呼吸。你做出了作品，已经很厉害了。我们一步一步来"
3. 如果学员实在无法继续，可以让他最后一个展示，先让其他同学继续
4. 梅杭俊可以在旁边单独安抚

## 讲师备忘

- 答辩全程保持积极、鼓励的语气。即使学员的项目很简单或有问题，也要找到值得肯定的地方
- 评分时记住：L1是"保底线"，大部分认真参与的学员都应该能达到L1。如果有学员连L1都达不到，先检查是不是评价标准过严
- 合议时如果有分歧，以"就高不就低"为原则。边界情况给学员更高的等级
- 公布结果时不要只念等级，每个人都配一句具体的肯定。避免"XXX，L1。下一个"这种冷冰冰的方式
- 课程结束后，考虑是否需要给学员建一个交流群，方便他们后续提问和互助
