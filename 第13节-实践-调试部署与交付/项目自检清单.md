# 项目自检清单

> 答辩前，对照这张清单检查你的项目。每完成一项就打勾。

---

## 基本要求（必须全部达到）

- [ ] **程序能运行**：在终端运行你的程序，没有报错，核心功能正常工作
- [ ] **能说清楚做什么**：用1-2句话说明"这个程序是做什么的、给谁用的"
- [ ] **能讲出一个问题**：开发过程中你遇到了什么问题？你是怎么发现和解决的？

---

## 加分项（做到了更好，没做到也没关系）

- [ ] **有使用说明**：写了 README 或者能口头说清楚怎么运行你的程序
- [ ] **别人能用**：让旁边的同学试试，不看你的屏幕、只按你的说明操作，能跑起来
- [ ] **能解释代码**：打开代码文件，能指出"这段是做什么的"
- [ ] **尝试了部署**：脚本有README / 网页能在浏览器打开 / 用了本地服务器或GitHub Pages

---

## 答辩准备

答辩的时候你需要做三件事：

**第一步：展示（2-3分钟）**
> 在你的电脑上运行/打开项目，展示核心功能

想想你要展示什么操作？按什么顺序展示？
- 我要展示的功能：____________________
- 展示步骤：____________________

**第二步：讲述（1分钟）**
> 用几句话说清楚这三个问题

- 这个项目做什么、给谁用？____________________
- 开发中遇到的一个关键问题是什么？____________________
- 你是怎么解决的？____________________

**第三步：回答提问（1-2分钟）**
> 讲师会问1-2个问题，不用紧张，就是聊天

讲师可能会问的问题类型：
- 你怎么想到做这个的？
- 这段代码是做什么的？
- 遇到的最大的坑是什么？
- 你是怎么跟 Claude Code 说的？
- 如果要加一个新功能，你会加什么？

---

**记住：答辩不是考试，是展示你的成长。做到什么程度就展示什么程度。**
