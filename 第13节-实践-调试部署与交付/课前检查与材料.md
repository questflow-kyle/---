# 第13节：课前检查与材料

## 课前15分钟检查清单

> 讲师在上课前15分钟到达教室，按以下顺序逐项检查。每完成一项打勾。

### 设备检查（5分钟）

- [ ] 投影仪开机，连接讲师电脑，画面正常
- [ ] 讲师电脑终端打开，字体大小调到后排能看清（建议18-20号字）
- [ ] 每台学员电脑开机，WSL 终端可正常打开
- [ ] 每台学员电脑 Python 可用：`python3 --version`
- [ ] 每台学员电脑 Claude Code 可用：`claude --version`
- [ ] 确认每台学员电脑上有第12节的项目文件（检查工作目录是否存在）

### 网络检查（3分钟）

- [ ] 教室 Wi-Fi 正常连接
- [ ] 在讲师电脑上测试 Claude Code 能正常响应：`claude "请说你好"`
- [ ] 在至少2台学员电脑上测试 Claude Code 能正常响应

### 教学材料确认（3分钟）

- [ ] **项目自检清单**：已打印，每人一张（见"项目自检清单.md"）
- [ ] **调试演示代码**：已保存到讲师电脑（见下方"备用代码"部分）
- [ ] **教案**：已打开或打印，放在手边随时查看

### 环境准备（2分钟）

- [ ] 在讲师电脑上准备好调试演示用的 bug 示例代码（见下方备用代码）
- [ ] 确认讲师电脑能运行 `python3 -m http.server`（部署演示用）

### 最终确认

- [ ] 教案已打开或打印，放在手边
- [ ] 项目自检清单已打印好
- [ ] 手机设为静音
- [ ] 确认上课时间，准备开始

## 备用代码

> 调试方法论演示时使用的 bug 示例代码。课前保存到讲师电脑。

### 调试演示代码（demo_bug.py）

```python
# demo_bug.py —— 故意留了bug的工资计算示例
workers = ["张三", "李四", "王五"]
wages = [3500, 4200, 3800]

for i in range(len(workers)):
    total = wages[i] + overtime[i]  # bug: overtime 没有定义
    print(f"{workers[i]} 的工资是 {total} 元")
```

## 常见报错速查表（可选打印/投影）

```
┌──────────────────────────────────────────────────────────────┐
│                    常见报错速查表                              │
│                                                              │
│  1. NameError: name 'xxx' is not defined                     │
│     意思：用了一个没有定义的变量名                               │
│     常见原因：变量名拼写错误，或者忘了创建这个变量                 │
│     怎么修：检查变量名拼写，确认变量在使用前已经定义               │
│                                                              │
│  2. SyntaxError: invalid syntax                              │
│     意思：代码写法不对，Python 看不懂                           │
│     常见原因：少了冒号、括号不配对、中英文标点混用                 │
│     怎么修：看报错指向的那一行，检查标点符号                      │
│                                                              │
│  3. TypeError: xxx                                           │
│     意思：数据类型不对                                         │
│     常见原因：字符串和数字混在一起计算                           │
│     怎么修：用 int() 或 float() 转换类型                       │
│                                                              │
│  4. IndexError: list index out of range                      │
│     意思：列表越界，访问了不存在的位置                           │
│     常见原因：循环次数多了，或者列表比你以为的短                   │
│     怎么修：检查列表长度和访问的下标                             │
│                                                              │
│  5. FileNotFoundError: No such file or directory             │
│     意思：找不到文件                                          │
│     常见原因：文件路径写错了，或者不在当前目录                     │
│     怎么修：用 ls 检查文件是否存在，用 pwd 确认当前目录           │
│                                                              │
│  6. IndentationError: unexpected indent                      │
│     意思：缩进有问题                                          │
│     常见原因：空格和Tab混用，或者缩进层级不对                     │
│     怎么修：统一用空格缩进（建议4个空格）                        │
│                                                              │
│  通用原则：先看最后一行（错误类型），再看倒数几行（出错位置）        │
└──────────────────────────────────────────────────────────────┘
```

## 备用方案

### 场景：Claude Code 不可用

如果 Claude Code 整堂课无法使用：

1. **调试演示**：不受影响，用备用代码手动演示
2. **项目完善**：学员可以手动修改代码。讲师帮学员读报错、定位问题，学员尝试自己改
3. **README撰写**：讲师在投影上展示一个 README 模板，学员参照着自己写

README 模板（投影展示）：
```
# 项目名称

## 这个程序做什么
（用1-2句话说清楚）

## 怎么运行
1. 打开终端
2. 进入项目目录：cd xxx
3. 运行：python3 xxx.py

## 需要什么环境
- Python 3
- （如果需要额外的包，写在这里）

## 使用说明
（程序会问你什么、你需要输入什么、会输出什么）
```

### 场景：个别学员电脑有问题

- 让该学员和旁边的同学共用一台电脑
- 尹伟优先协助排查该电脑问题

### 场景：大部分学员项目还没跑通

- 压缩部署概念到1分钟（只说一句"写README就行"）
- 把省下来的时间全部给项目完善
- 讲师和助教全力巡场帮助调试
