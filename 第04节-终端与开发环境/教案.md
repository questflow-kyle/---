# 第4节：终端与开发环境 —— 剧本式教案

## 课时信息

| 项目 | 内容 |
|:---|:---|
| 所属阶段 | 基础课（第4节/共14节） |
| 主讲 | 沈林伟（终端与Claude Code）、尹伟（硬件快速介绍） |
| 时长 | 45 分钟 |
| 班级规模 | 5-10 人 |
| 前置课程 | 第1-3节（搜索、AI对话、办公软件） |

## 教学目标

学完本节课，学员能够：
1. 认识CPU、内存、硬盘、主板的实物，说出各自的基本功能
2. 在WSL终端中执行 `ls`、`cd`、`pwd`、`mkdir` 命令
3. 确认 Python 环境可用（`python3 --version`）
4. 启动 Claude Code 并完成一次对话

## 本节课的核心定位

> **这是转折课。** 前三节学员都在用图形界面（浏览器、Word、Excel），从这节课开始，他们要面对一个黑色的、只有文字的屏幕。
>
> 这节课的 **第一目标不是教会命令，而是消除恐惧**。学员需要建立一个信念："终端不可怕，打错了不会弄坏电脑，我可以学会。"
>
> 所有教学设计都围绕这个目标：先看到它能工作，再理解它为什么工作。

### 设计说明

> 从课程体系的角度看，本节课是从"信息素养+AI对话+办公能力"过渡到"编程+开发"的桥梁。前三节建立的能力（渠道意识、三问法、提问三要素、信但验、人机协作模式）在本节课开始向开发领域迁移。
>
> 有机整合体现在两个关键连接：
> 1. **Claude Code是网页版AI的升级**：学员在第2-3节用网页版AI写文字、写公式，已经建立了"信但验"的习惯。Claude Code做的事情更强大——它不光能聊天，还能直接在电脑上创建文件、写代码、运行代码。能力更强，验证就更重要。本节课需要自然地点明这个联系。
> 2. **人机协作模式的延续**：第3节建立的"AI生成→人验证"模式，到第5节会变成"AI写代码→人读懂和运行"。本节课是中间环节——让学员熟悉终端这个"AI住的地方"，为后续协作做准备。

## 时间分配

| 环节 | 时长 | 主讲 | 内容 |

## 环节一：学员回顾第3节（5分钟）

> **设计意图**：从本节课开始，每节课的前5分钟由学员完成回顾。这是"同伴教学"机制的起点——练习"把学到的东西讲给别人听"的能力。第一次做，需要讲师多引导。

### 讲师台词（沈林伟）

> "大家好。从今天开始，每节课一上来，我们会请一位同学回顾上节课的内容。不用紧张，不是考试，就是说说你记住的东西。这个习惯以后每节课都会有。"
>
> "谁愿意来说说？上节课梅杭俊老师讲了什么？"

### 操作

1. 等几秒看有没有人主动举手
2. 如果没人举手，讲师点名——**优先选上节课表现积极的学员**，降低压力
3. 让学员说2-3分钟，不打断
4. 学员说完后，讲师补充确认以下要点被提到：
   - Word/WPS 基本排版（字体、段落、导出PDF）
   - Excel 基本操作（SUM、AVERAGE公式）
   - AI协作模式：AI生成内容 + 自己排版检查（信但验）

### 讲师台词（补充时）

> "说得很好，谢谢。我补充一个点：上节课最重要的是那个模式——**AI帮你生成内容，你来排版和检查**。还有一个原则——**信但验**。这两样东西后面会一直用到，而且会越来越重要。"
>
> "好，今天我们要做一件不一样的事情。前三节课我们都是用鼠标点点点，今天开始，我们要用键盘跟电脑'说话'。"

### 如果学员完全回顾不出来

不要批评。讲师自己用30秒快速过一遍要点。重点是让学员感受到"你们可以当老师"，不是考试。

### 如果是第一次做回顾，学员明显紧张

讲师可以这样引导：

> "没关系，就说一个你记住的东西就行。比如上节课用了什么软件？"

## 环节二：硬件快速认识（5分钟）—— 尹伟主讲

> **设计意图**：让学员认识电脑的核心零件。不求深入理解，只要求"看到实物能叫出名字、说出功能"。5分钟，点到为止。

### 尹伟台词

> "在进入终端之前，我花5分钟带大家认识一下电脑里面长什么样。林伟后面要讲的东西都是软件层面的，我先让大家看看硬件层面。"

### 操作

1. 拿出已拆开的台式机（或者拆好的硬件部件），放在讲台上
2. 逐一展示四个核心部件，每个部件30秒-1分钟

### 部件一：CPU

**尹伟台词**：

> "这个小方块叫CPU，中文叫处理器。它是电脑的**大脑**，所有的计算都在这里面完成。你打开一个网页、播放一个视频，都是CPU在干活。"

（拿起来让前排学员看一眼，不需要传一圈，节省时间）

### 部件二：内存（RAM）

**尹伟台词**：

> "这个长条叫内存条，英文叫RAM。它是电脑的**工作台**——你正在用的东西就放在这上面。比如你打开了Word，Word的内容就暂时存在内存里。但是注意，**断电就没了**。所以你们上节课学的Ctrl+S保存很重要，保存了才会写到硬盘里。"

（拿起来展示）

### 部件三：硬盘（SSD/HDD）

**尹伟台词**：

> "这个叫硬盘，它是电脑的**仓库**。你的文件、照片、软件都存在这里面。跟内存不一样，硬盘断电不会丢东西。现在的新电脑基本用这种小一点的固态硬盘（SSD），速度快。老电脑用那种厚一点的机械硬盘（HDD）。"

### 部件四：主板

**尹伟台词**：

> "这一大块板子叫主板。它是电脑的**骨架**——CPU插在上面，内存插在上面，硬盘接在上面，所有零件都通过主板连在一起，互相通信。"

### 快速总结

**尹伟台词**：

> "总结一下，记四个东西就够了：
>
> | 部件 | 比喻 | 功能 |
> |:---:|:---:|:---|
> | CPU | 大脑 | 负责计算 |
> | 内存 | 工作台 | 临时存放正在用的东西，断电就没了 |
> | 硬盘 | 仓库 | 长期存放文件，断电不丢 |
> | 主板 | 骨架 | 把所有零件连在一起 |
>
> "不用死记，有个印象就行。以后电脑出问题了至少知道可能是哪一块的问题。好，硬件就到这里，交给林伟。"

### 时间控制

这个环节**严格控制在5分钟**。如果尹伟发现自己说多了，看到4分钟直接跳到总结表。不要展开讨论，不要让学员提问（学员的问题课后答）。

---

## 环节三：终端基础命令（15分钟）—— 沈林伟主讲

> **设计意图**：这是本节课的核心环节。学员第一次面对终端黑屏，心理压力大。教学策略是：
> 1. 先类比，让学员明白终端是什么
> 2. 一个命令一个命令教，每个命令讲完立刻让学员跟着做
> 3. 反复强调"打错了不会坏"
>
> **重要原则**：讲师投影上的每一步操作，学员都要在自己电脑上同步做。不是"先看我演示再你做"，而是"我做一步你做一步"。

### 3.1 打开终端 + 消除恐惧（3分钟）

#### 讲师台词

> "好，现在开始今天的重头戏。大家看我的屏幕。"

**讲师操作**：打开 Windows Terminal 或 WSL 终端（投影展示）

> "这个黑色的窗口叫**终端**，英文叫 Terminal。它看起来跟你们之前用的东西完全不一样——没有按钮，没有菜单，只有一个闪烁的光标等你打字。"
>
> "很多人第一次看到终端会害怕：'这是什么鬼？我会不会打错什么把电脑弄坏？'**不会。** 我今天先把话撂在这里：**在终端里打错命令，最多就是它告诉你'我不认识这个命令'，然后什么都不会发生。** 你的电脑不会爆炸，你的文件不会消失。放轻松。"
>
> "终端其实就是**用打字代替鼠标点击**。你用鼠标双击文件夹打开它，在终端里你打一个命令也能打开它。做的事情一模一样，只是方式不同。"
>
> "为什么要学终端？因为后面我们要用的工具——Claude Code——它就住在终端里。你要跟它说话，必须先会在终端里走路。"

#### 让学员打开终端

> "现在请大家在自己电脑上打开终端。方法是：点击桌面左下角的开始菜单，找到 **Ubuntu** 或者 **Windows Terminal**，点击打开。"

**讲师和助教巡场**，确认每个人都打开了终端窗口。

> "打开了的举手？"（确认全部打开）
>
> "好。大家看到屏幕上应该有一行字，最后有一个闪烁的光标。那行字叫**提示符**，它告诉你'我准备好了，等你输入命令'。就像服务员站在你面前问'请问你要点什么？'"

### 3.2 第一个命令：pwd（2分钟）

#### 讲师台词

> "我们来输入第一个命令。请大家跟着我打字。输入三个字母，小写：p-w-d，然后按回车。"

**讲师操作**（投影）：

```bash
pwd
```

**预期输出**：

```
/home/你的用户名
```

#### 讲师台词

> "大家看到了什么？它输出了一行路径，类似 `/home/ubuntu` 或者 `/home/你的用户名`。"
>
> "`pwd` 是什么意思？**Print Working Directory**——显示你当前在哪个位置。就像你在商场里，看到一块牌子写着'您在此处'，`pwd` 就是那块牌子。"
>
> "每次你不知道自己在哪里了，就输入 `pwd`，它会告诉你。"

**确认学员都做了**：

> "大家都打了 `pwd` 了吗？看到路径了吗？如果看到了路径，恭喜你，你已经成功执行了你的第一个终端命令。"

### 3.3 第二个命令：ls（2分钟）

#### 讲师台词

> "第二个命令，两个字母：l-s，小写，按回车。"

**讲师操作**（投影）：

```bash
ls
```

**预期输出**：可能是空的，也可能列出一些文件夹名称。

#### 讲师台词

> "`ls` 是什么意思？**List**——列出当前目录下有什么东西。就像你打开一个文件夹，看到里面有哪些文件和子文件夹。"
>
> "如果你看到一些名字，那就是当前目录下的文件或文件夹。如果什么都没有，说明当前目录是空的，正常。"

**让学员做**：

> "大家都输入 `ls` 了吗？看到结果了吗？"

### 3.4 第三个命令：mkdir（3分钟）

#### 讲师台词

> "现在我们来创建一个文件夹。这个命令叫 `mkdir`，意思是 **Make Directory**——创建目录。大家跟我打："

**讲师操作**（投影）：

```bash
mkdir learning
```

> "注意，`mkdir` 后面跟一个空格，然后是文件夹的名字，这里我们叫它 `learning`。按回车。"

**等学员操作完。**

> "好，它什么都没输出对吧？这就对了。**终端里，没消息就是好消息。** 没有输出说明命令成功执行了。如果失败了，它会报错告诉你。"
>
> "我们怎么确认文件夹创建成功了？用刚才学的命令——"

```bash
ls
```

> "看到 `learning` 了吗？出现了吧？说明你刚才成功创建了一个文件夹。"

**让学员操作，确认每个人都看到了 `learning`。**

> **如果有学员输错了（比如拼写错了）**：不要批评，走过去轻声说"没关系，按上方向键可以看到之前的命令，改一下重新按回车就行。"

### 3.5 第四个命令：cd（3分钟）

#### 讲师台词

> "最后一个命令：`cd`，意思是 **Change Directory**——切换目录。就像你在文件管理器里双击进入一个文件夹。大家跟我打："

**讲师操作**（投影）：

```bash
cd learning
```

> "`cd` 后面跟一个空格，然后是文件夹的名字。按回车。"

**等学员操作完。**

> "好，它又没输出对吧？没消息就是好消息。但这次你的位置变了。怎么确认？用 `pwd`。"

```bash
pwd
```

**预期输出**：

```
/home/你的用户名/learning
```

> "看到了吗？路径最后多了一个 `/learning`，说明你已经'走进'了 learning 这个文件夹里面。"
>
> "再输入 `ls` 看看里面有什么。"

```bash
ls
```

> "空的对吧？因为我们刚创建这个文件夹，里面还什么都没有。"
>
> "现在我们再学一个小技巧——怎么从文件夹里退出来？"

```bash
cd ..
```

> "`cd` 后面跟两个点 `..`，意思是'回到上一层目录'。就像你在文件管理器里点那个向上的箭头。"

```bash
pwd
```

> "看，又回到刚才的位置了。"

### 3.6 小结（2分钟）

#### 讲师台词

> "好，停下来总结一下。我们刚才学了四个命令，一共花了不到十分钟。"

**投影展示**（建议全程保留在屏幕一角）：

```
终端命令速查：
pwd           你在哪里（显示当前位置）
ls            这里有什么（列出文件和文件夹）
mkdir 名字    创建文件夹
cd 名字       进入文件夹
cd ..         回到上一层
```

> "这四个命令就是终端里'走路'的方法。你用 `pwd` 看自己在哪，用 `ls` 看周围有什么，用 `cd` 走到下一个地方，用 `mkdir` 建一个新地方。就像在迷宫里拿着地图走路一样。"
>
> "我再强调一遍：**打错了不会坏。** 如果你输错了命令，终端会说 `command not found`——'我不认识这个命令'——然后什么都不做，等你重新输入。所以放心大胆地打字。"

## 环节四：Claude Code 初体验（10分钟）

> **设计意图**：让学员体验"在终端里跟AI说话"的全新感觉。这个环节只做最简单的体验——启动Claude Code、问一个问题、看到回答、退出。不写代码，不做复杂操作。让学员建立"这个工具我能用"的信心。

### 4.1 确认 Python 可用（2分钟）

#### 讲师台词

> "在启动 Claude Code 之前，我们先确认一下 Python 环境。Python 是一种编程语言，后面我们写代码要用到它。现在只需要确认它装好了就行。"
>
> "请大家先回到 home 目录，然后输入以下命令："

**讲师操作**（投影）：

```bash
cd ~
```

> "`cd ~` 是回到你的'家目录'的意思，就是你每次打开终端的起始位置。"

```bash
python3 --version
```

**预期输出**：

```
Python 3.10.12
```

（版本号可能不同，只要输出了版本号就算正常。）

> "如果你看到了 `Python 3.xxx`，说明 Python 已经装好了。如果你看到了'command not found'或者报错，举手，我来帮你。"

**讲师和助教巡场**，帮助有问题的学员。课前检查应已确保所有电脑Python正常，此处应无问题。

> "好，Python 就绪。后面我们写代码的时候会用到它。现在我们来启动今天最重要的工具。"

### 4.2 启动 Claude Code（3分钟）

#### 讲师台词

> "现在进入今天最激动人心的部分。大家还记得前几节课我们用的网页版AI吗？今天我们要用一个更厉害的版本——**Claude Code**。它不是在浏览器里的，它就住在终端里。"
>
> "它厉害在哪里？网页版AI只能跟你聊天。但 Claude Code 不光能跟你聊天，它还**能直接在你的电脑上创建文件、写代码、运行代码**。后面的课我们都会用它来写程序。"
>
> "但是——还记得我们的原则吗？**信但验**。网页版AI会编东西，Claude Code 也一样——它本质上是同一个AI，只是多了操作电脑的能力。能力更强，验证就更重要。这个后面的课会反复练到，今天先有个印象。"
>
> "怎么启动它？很简单。先进入我们刚才创建的文件夹："

**讲师操作**（投影）：

```bash
cd ~/learning
```

> "然后输入这个命令："

```bash
claude
```

> "就一个词，`claude`，按回车。"

**等待 Claude Code 启动。** 启动后会显示一个交互界面。

> "大家看到了吗？界面变了。这就是 Claude Code 的对话界面。注意看底部的输入框——你在那里打字，它就会回答你。跟网页版AI一样，只不过现在是在终端里面。"

**如果有学员 Claude Code 启动失败**：
- 第一时间让助教过去看
- 如果2分钟内能修好就修
- 如果修不好，让该学员先看旁边同学的屏幕，课后解决

### 4.3 跟 Claude Code 说第一句话（3分钟）

#### 讲师台词

> "好，现在我们来跟 Claude Code 说第一句话。大家跟着我输入——这句话很简单，就一个问题："

**讲师操作**（在 Claude Code 中输入）：

```
用一句话解释什么是终端
```

**等待 Claude Code 回答。**

> "大家看它的回答。是不是用中文回答的？而且说得还挺清楚的。"
>
> "来，我们再问一个："

```
终端里的 ls 命令是做什么用的？
```

**等待回答。**

> "看到了吗？它不光回答了 ls 的功能，可能还给了例子。这就是 Claude Code——你在终端里随时可以问它问题。以后你忘了某个命令怎么用，直接问它就行。"

### 4.4 退出 Claude Code（2分钟）

#### 讲师台词

> "好，现在我们先退出 Claude Code。怎么退？输入 `/exit`，注意前面有个斜杠。"

**讲师操作**：

```
/exit
```

> "好，又回到了普通的终端界面。`/exit` 就是退出 Claude Code 的命令，记住它。"
>

## 常见问题与应对（FAQ）

### Q1："这个黑屏看着好吓人，我会不会不小心把电脑弄坏？"

> **这是最重要的问题，必须正面回应。**
>
> **回答**："不会。你在终端里能打的命令，99.9%都是安全的。最坏的情况就是它说'command not found'——我不认识这个命令——然后什么都不发生。就像你在手机上打错一个字发出去了，对方看不懂而已，手机不会坏。我们今天学的 `pwd`、`ls`、`cd`、`mkdir` 这些命令都是只读或者创建文件夹的操作，完全安全。"

### Q2："我打了命令但什么都没反应，是不是卡了？"

> **回答**："大概率是命令执行成功了。终端里有个规矩：**没有消息就是好消息。** 比如你 `mkdir` 创建文件夹，如果成功了它就什么都不说。你输入 `ls` 确认一下就知道了。只有出错的时候它才会说话。"

### Q3："我输入了命令，它说 `command not found`，是不是我打错了？"

> **回答**："对，这就是说终端不认识你输入的命令。最常见的原因：
> 1. **拼写错了**——检查一下，`ls` 是小写L和S，不是数字1和5
> 2. **多了空格或少了空格**——`mkdir learning` 中间有一个空格
> 3. **中英文输入法没切换**——终端里所有命令和文件名都是英文的
>
> 看看是哪个情况，改正后重新输入就行。"

### Q4："怎么清屏？屏幕上东西太多了看着乱。"

> **回答**："输入 `clear`，按回车，屏幕就干净了。你之前输入的东西不会丢，只是画面清理了。"

### Q5："我输到一半发现打错了，怎么办？"

> **回答**："按 `Ctrl+C`，取消当前输入，重新来。或者用退格键（Backspace）一个一个字符删。按上方向键可以调出之前打过的命令，改一下再按回车。"

### Q6："Claude Code 和之前用的网页版AI有什么区别？"

> **回答**："网页版AI只能跟你聊天——你问它问题，它回答，就这样。Claude Code 也能聊天，但它更强大：它能直接在你的电脑上创建文件、写代码、运行代码、修改文件。下节课你就会看到这个区别了。"

### Q7："ls 和 pwd 这些命令是英文缩写吗？必须记住吗？"

> **回答**："对，都是英文缩写。`pwd` 是 Print Working Directory，`ls` 是 List，`cd` 是 Change Directory，`mkdir` 是 Make Directory。不需要死记，用几次就自然记住了。忘了就看速查表，或者直接问 Claude Code——它会告诉你的。"

### Q8："学员问了一个你不知道的终端问题"

> **回答**："好问题。我不确定，我们可以直接问 Claude Code。"然后在投影上演示启动 Claude Code 问这个问题——这本身就是教学。

## 时间管理备注

### 正常节奏下的时间分配

| 环节 | 时长 | 累计 |
|:---|:---:|:---:|
| 学员回顾 | 5分钟 | 0:05 |
| 硬件快速认识 | 5分钟 | 0:10 |
| 终端基础命令 | 15分钟 | 0:25 |
| Claude Code 初体验 | 10分钟 | 0:35 |
| 学员练习 | 8分钟 | 0:43 |
| 课后小结 | 2分钟 | 0:45 |

### 如果时间超了，按优先级砍

1. **第一刀：缩短学员回顾**。如果学员回顾用了4-5分钟，讲师补充只用一句话带过，不再展开。省下2分钟。
2. **第二刀：砍硬件总结表**。四个部件展示完直接说"就这四个，有印象就行"，不投影表格。省下1分钟。
3. **第三刀：砍练习任务一中的步骤6-7**（在 mytest 里再创建 abc）。只做到进入 mytest 并 pwd 确认即可。省下2分钟。
4. **第四刀：缩短 Claude Code 对话**。只问一个问题，不问两个。省下1-2分钟。

**绝对不砍的内容**：
- 终端命令的讲解和同步操作（环节三）——这是本课核心
- Claude Code 的启动和退出——下节课必须会
- 练习任务二中的 `python3 --version` ——下节课前置依赖

### 如果进度比预期快（多出5分钟）

追加一个小活动：

> "我们多练一个东西。大家启动 Claude Code，问它：'请列出5个最常用的终端命令并用一句话解释每个命令。'看看它给你什么建议。"
