# 第12节：实践——自选项目

## 概要

> 本节学员第一次从"给定任务"变为"自己定义问题"。这是最接近真实开发的状态——自己发现需求、自己描述、用AI实现。讲师角色从"带着做"转为"教练巡场"。项目成果将用于第14节的答辩考核。

## 教案

### 课时信息

- 所属阶段：实践课（第4节/共5节）
- 主讲：沈林伟
- 辅助：梅杭俊、尹伟（全体讲师巡回指导）
- 时长：45 分钟

### 教学目标

学完本节课，学员能够：
1. 从自身生活或工作中找到一个可以用程序解决的问题
2. 把模糊的需求拆解成具体的功能描述
3. 用Claude Code开始开发自己的项目
4. 项目有初步可运行的版本

### 任务说明

#### 核心任务：解决一个自己的真实问题

学员自选一个项目方向，用Claude Code开发。可以是脚本类（自动化工具）或应用类（网页/小工具），由学员根据自身兴趣和能力决定。

#### 项目方向参考（没有想法的学员可从中选择）

| 方向 | 示例 |
|:---|:---|
| 生活工具 | 个人记账本、日程提醒器、生活费分摊计算器 |
| 工作工具 | 考勤统计器、工资核算工具、排班表生成器 |
| 学习工具 | 单词记忆卡片、打字练习器、知识问答小游戏 |
| 信息工具 | 天气查询页面、汇率换算器、倒计时网页 |

#### 分层要求

| 等级 | 完成标准 |
|:---:|:---|
| L1 能用 | 能用中文清晰描述需求，用Claude Code跑出初步可运行的版本 |
| L2 能调 | L1 + 能在初版基础上修改功能或修复问题；能解释代码的主要逻辑 |
| L3 能交付 | L2 + 项目功能完整、有使用说明、能交给别人使用 |

### 教学流程

| 时间 | 内容 |
|:---:|:---|
| 0-5分钟 | 讲师说明任务要求和项目方向参考 |
| 5-12分钟 | 学员确定项目方向，用中文写出需求描述（至少3句话说清楚"做什么、给谁用、要实现哪些功能"） |
| 12-15分钟 | 快速轮流：每人30秒说出自己要做什么，讲师给出方向建议 |
| 15-40分钟 | 独立开发时间，全体讲师巡回指导 |
| 40-45分钟 | 进度检查：每人一句话汇报当前状态（跑通了/卡在哪了/下一步做什么） |

### 教学提示

- "自选项目"对零基础学员可能有压力，准备好项目参考清单，让没想法的人有得选
- 需求描述是本节的关键环节——描述清楚了，Claude Code才能写对。帮学员把"我想做一个xxx"变成"输入是什么、处理什么、输出什么"
- 全体讲师巡回：沈林伟负责技术指导，梅杭俊帮学员梳理需求，尹伟辅助环境问题
- 项目不要求本节完成，第13节还有时间调试和完善。本节重点是"启动并跑出初版"
- 提醒学员：这个项目就是第14节答辩的作品
