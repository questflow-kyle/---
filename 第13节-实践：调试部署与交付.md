# 第13节：实践——调试、部署与交付

## 概要

> 实践课最后一节，目标是把第12节启动的自选项目打磨到"能交付"的状态。前半段集中讲调试方法论和部署基础概念，后半段学员自己完善项目。这是考核前最后的开发时间。
>
> **待补充**：

> - [ ] 准备常见报错速查表
> - [ ] 设计项目自检清单（学员答辩前对照检查）

## 教案

### 课时信息

- 所属阶段：实践课（第5节/共5节）
- 主讲：沈林伟
- 辅助：尹伟
- 时长：45 分钟

### 教学目标

学完本节课，学员能够：
1. 掌握基本的调试思路（读报错→缩小范围→定位问题→修复验证）
2. 理解"部署"的概念（让程序在目标环境下跑起来）
3. 把自选项目完善到可展示的状态
4. 准备好第14节答辩的内容

### 内容要点

#### 一、调试方法论（10分钟）

- 报错不可怕，它是程序在告诉你哪里有问题
- 调试四步法：
  1. **读报错**：看最后一行，它通常直接说了是什么错
  2. **缩小范围**：是代码逻辑错了？还是环境问题？还是输入数据问题？
  3. **定位问题**：找到出错的那一行或那一段
  4. **修复验证**：改了之后重新运行，确认问题解决
- 用Claude Code调试的正确姿势：
  - 不要只说"报错了帮我修"
  - 要说"第X行报了XX错误，我觉得可能是XX原因，帮我看看"
  - 给Claude Code看到完整的报错信息

#### 二、部署基础概念（5分钟）

- 部署 = 让你的程序在目标环境里跑起来
- 脚本类项目：确保别人的电脑上也能运行（依赖说明、运行方法）
- 网页类项目：让别人能通过浏览器访问（本地打开 / 放到服务器上）
- 最简部署：
  - 脚本：写一份README说明"怎么运行这个程序"
  - 网页：用 `python3 -m http.server` 启动本地服务器；或部署到GitHub Pages
- 交付标准：别人按你的说明操作，能把程序跑起来

#### 三、项目完善时间（25分钟）

- 学员根据自身情况选择重点：
  - 项目还没跑通 → 优先调试，确保基本功能可用
  - 项目基本可用 → 添加功能或优化体验
  - 项目已完善 → 写使用说明、尝试部署
- 讲师巡回指导

#### 四、答辩准备（5分钟）

- 项目自检清单：
  - [ ] 程序能运行，核心功能正常
  - [ ] 能用1-2句话说清楚"这个程序做什么"
  - [ ] 能讲出开发过程中遇到的一个问题和解决方法
  - [ ] （加分）有使用说明或能展示给别人用
- 下节课答辩流程预告

### 教学提示

- 调试方法论是区分L1和L2的关键——L1只是让AI修，L2能自己定位方向再让AI修
- 部署概念点到即止，对大部分学员来说"本地能跑+有使用说明"就是合格的交付
- 冲L3的学员可以引导尝试GitHub Pages或简单服务器部署，尹伟可协助
- 最后5分钟的答辩准备很重要，帮学员提前组织思路，避免第14节临场紧张
