# 第9节：实践——自动化脚本

## 概要

> 实践课第一节，从最直觉的需求切入：把电脑里乱七八糟的文件自动整理好。所有学员做同一个任务，通过完成深度体现L1/L2/L3的差异。本节全程使用Claude Code。
>
> **待补充**：
> - [ ] 准备练习文件夹（30-50个不同类型的文件：jpg/png/txt/pdf/docx/csv等）
> - [ ] 确认每台电脑Claude Code可用

---

## 教案

### 课时信息

- 所属阶段：实践课（第1节/共5节）
- 主讲：沈林伟
- 辅助：梅杭俊
- 时长：45 分钟

### 教学目标

学完本节课，学员能够：
1. 用Claude Code独立完成一个文件自动整理脚本
2. 理解脚本的基本逻辑（扫描→判断→移动→报告）
3. 根据自己的需求修改整理规则

### 任务说明

#### 核心任务：批量文件整理器

给定一个混乱的文件夹（包含图片、文档、表格、PDF等各种文件），写一个脚本自动按类型分类到子文件夹，并生成整理报告。

#### 分层要求

| 等级 | 完成标准 |
|:---:|:---|
| L1 能用 | 用Claude Code描述需求，跑通脚本，文件成功分类到子文件夹 |
| L2 能调 | L1 + 修改分类规则（如按日期分类而非按类型）；能解释脚本中循环和条件判断的作用 |
| L3 能交付 | L2 + 添加功能（如重复文件检测、整理日志、命令行参数支持）；脚本可以给别人直接使用 |

### 教学流程

| 时间 | 内容 |
|:---:|:---|
| 0-5分钟 | 讲师演示任务目标：展示乱文件夹 → 运行脚本 → 展示整理结果 |
| 5-10分钟 | 引导学员拆解需求：用中文写出"程序要做哪几步" |
| 10-35分钟 | 学员用Claude Code独立开发，讲师巡回指导 |
| 35-45分钟 | 2-3位学员展示成果，讲师点评 |

### 教学提示

- 开始前让学员先用中文写出需求（"扫描文件夹→判断文件类型→移到对应子文件夹→生成报告"），再交给Claude Code
- 巡回时重点观察：学员是否只是让AI跑通了（L1），还是在尝试理解和修改（L2/L3）
- 鼓励L1的学员至少尝试改一个参数（如子文件夹命名）
- 遇到报错时引导学员先自己读错误信息，再让Claude Code修复

### 所需材料/设备

- [ ] 每人一台电脑（WSL + Python + Claude Code）
- [ ] 预制的混乱文件夹（30-50个不同类型文件，每台电脑一份）
- [ ] 投影仪
