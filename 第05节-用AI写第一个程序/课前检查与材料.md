## 一、环境检查（课前30分钟，逐台电脑）

> **这是本节课最关键的准备工作。** 上节课刚装好环境，本节是第一次真正使用 Claude Code 写代码。如果环境有问题，课上排障会严重占用教学时间。

### 每台电脑检查清单

对每一台学员电脑执行以下操作：

#### 1. WSL 终端能正常打开
- [ ] 打开 Windows Terminal 或 WSL 终端
- [ ] 输入 `pwd`，确认能输出路径（说明终端正常工作）

#### 2. Python 环境可用
- [ ] 输入 `python3 --version`，确认输出版本号（如 `Python 3.10.x` 或更高）
- [ ] 输入 `python3 -c "print('测试通过')"` ，确认输出"测试通过"

#### 3. Claude Code 可用
- [ ] 输入 `claude --version`，确认输出版本号
- [ ] 输入 `claude`，确认能正常启动（看到 Claude Code 交互界面）
- [ ] 在 Claude Code 中输入一句话测试（如"你好"），确认能收到回复
- [ ] 输入 `/exit` 退出 Claude Code

> **重要**：步骤3是最容易出问题的地方。如果 Claude Code 启动后报错（如 API key 无效、网络不通），必须在课前解决。

#### 4. API 配额检查
- [ ] 确认 API key 配置正确（课前跟 Claude Code 对话几轮，没有报配额错误）
- [ ] 预估本节课用量：每位学员约 5-8 次对话，每次对话约消耗少量 token。10位学员约 50-80 次对话
- [ ] 如果使用共享 API key，确认余额充足

#### 5. 工作目录准备
- [ ] 在每台电脑上创建工作目录：
  ```bash
  mkdir -p ~/learning
  ```
- [ ] 确认目录创建成功：`ls ~/learning`

## 二、讲师自己的电脑（投影用）

- [ ] 投影仪连接正常，投影画面清晰
- [ ] 终端字体足够大，后排学员能看清（建议字号 18-22）

  3. 让它写 `hello.py`
  4. 退出，运行 `python3 hello.py`
  5. 再启动 `claude`，让它加 `input` 功能

### 投影内容准备

课上需要投影的内容，建议提前准备好（可以是文本文件或笔记）：

**投影1：编程方法（环节二用）**
```
编程 = 给电脑写操作手册

我们的方法：
你说需求 → AI写代码 → 你运行 → 你读懂 → 你修改
```

**投影2：Claude Code 演示提示词（环节三用）**
```
提示词1：
写一个Python程序 hello.py，功能是打印一句"你好，欢迎来到编程的世界！"

提示词2：
修改 hello.py，让它先问用户叫什么名字，然后用用户的名字打招呼。
比如用户输入"小明"，程序输出"小明，你好！欢迎来到编程的世界！"
```

**投影3：三个核心概念（环节四用）**
```
变量 variable  = 贴了标签的盒子，用来存东西
print          = 在屏幕上显示内容
input          = 等用户输入
```

**投影4：练习任务提示词（环节五用）**
```
练习提示词：
写一个Python程序 intro.py，功能是：
1. 问用户叫什么名字
2. 问用户来自哪里
3. 问用户最喜欢吃什么
4. 把这些信息组合起来，输出一段自我介绍

修改选择A：
修改 intro.py，把最后的问候语改成更有趣的版本

修改选择B：
修改 intro.py，再加一个问题，问用户的爱好是什么，然后也加到自我介绍里
```

**投影5：常用命令速查（全程保留）**
```
cd ~/learning          进入工作目录
claude                 启动 Claude Code
/exit                  退出 Claude Code
python3 文件名.py      运行程序
cat 文件名.py          查看代码内容
pwd                    查看当前在哪个目录
ls                     查看目录下有什么文件
```

## 四、备用方案

### 备用方案A：个别电脑 Claude Code 无法使用

**症状**：`claude` 命令报错、API key 无效、网络不通等

**应对**：
1. 先尝试快速修复（2分钟内能解决就修）
2. 如果修不好，该学员与旁边同学共用一台电脑（两人一组，一人操作一人观察，轮流）
3. 课后单独帮该学员修复环境

### 备用方案B：全部电脑 Claude Code 无法使用（如网络中断、API服务故障）

**这是最坏情况，概率低但必须有预案。**

**应对**：
1. 切换到网页版 Claude（https://claude.ai）或其他可用的网页版AI
2. 在网页版AI中输入同样的提示词，生成代码
3. 手动将代码复制到文件中：
   ```bash
   nano hello.py
   ```
   （将AI生成的代码粘贴进去，`Ctrl+O` 保存，`Ctrl+X` 退出）
4. 教学流程其余部分不变：运行、读代码、修改
5. 向学员解释："今天网络有点问题，我们换个方式，但方法是一样的——AI写代码，你运行。"

> **注意**：用网页版AI + 手动复制的方式体验会差一些（多了复制粘贴步骤），但核心教学目标不受影响。

### 备用方案C：个别学员打字极慢

**应对**：
1. 梅杭俊帮忙打字，但让学员口述要输入什么
2. 可以让学员看着投影上的提示词，逐字输入（不要求速度）
3. 课后建议学员利用业余时间练习打字（可推荐打字练习网站）

### 备用方案D：课堂时间不够

**应对**：
- 参见教案中的"时间应急方案"
- 底线：确保每位学员至少看到了完整演示 + 自己运行过一次程序
- 修改代码的练习可以放到下节课开头
