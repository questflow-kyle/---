# 第2节：AI对话与提问技巧 —— 课前检查与材料

## 课前15分钟检查清单

> 讲师在上课前15分钟到达教室，按以下顺序逐项检查。每完成一项打勾。

### 硬件与网络（5分钟）

- [ ] **投影仪/大屏幕**：开机，连接讲师电脑，画面正常显示
- [ ] **讲师电脑**：打开浏览器，确认能正常上网
- [ ] **学员电脑**：逐台确认已开机、已联网、浏览器可用
  - 逐台打开AI工具网页（如 claude.ai），确认能正常加载
  - 如果有电脑无法联网 → 见下方"备用方案A"
  - 如果有电脑无法访问AI工具 → 见下方"备用方案B"
- [ ] **网络速度测试**：在讲师电脑上向AI发送一句话，确认回复在10秒内开始生成
  - 如果AI响应缓慢 → 见下方"备用方案C"

### AI工具准备（5分钟）

- [ ] **讲师电脑AI工具**：已登录AI工具（网页版），确认能正常对话
- [ ] **学员账号**：确认学员使用的AI账号/登录方式
  - 如果用统一账号：账号密码已写在黑板/白板上
  - 如果用个人账号：确认每位学员已注册并能登录
- [ ] **AI幻觉演示预测试**（最关键的一步）：
  - 在AI中输入教案中的幻觉案例一（劳动法第九十三条），确认AI会编造内容
  - 如果AI回答正确（说该条不存在或正确引用），立即测试备用案例（成都市金牛区劳动保障服务中心地址电话）
  - 记录哪个案例可用：□ 案例一（法律条文） □ 案例二（地址电话）
  - **如果两个案例都失败**：临时准备一个新案例（比如问AI一个虚构的成都本地餐馆名称和地址，然后搜索验证不存在）
- [ ] **演示对话清空**：讲师AI工具中清空之前的对话记录，确保演示时界面干净

### 教学材料确认（3分钟）

- [ ] **教案**：已打开或打印，放在手边可随时查看
- [ ] **提问对比卡**：已打印，份数 = 学员人数（见本文档"提问对比卡"部分）
- [ ] **投影内容准备**：以下内容可以直接从教案投影，确认能清晰展示
  - AI"擅长/做不到"对比表格
  - 提问三要素框图
  - 三问法应用于AI（幻觉环节使用）
  - "信但验"原则板书
  - 三个练习任务的任务说明

### 环境布置（2分钟）

- [ ] **座位安排**：学员之间留有适当间距，讲师能在走道中巡场
- [ ] **投影画面**：字体大小从最后一排能看清（打开AI对话界面，问最后排的人能不能看清对话内容）
- [ ] **讲师站位**：站在投影屏幕旁边，不挡住画面，同时能操作电脑

### 最终确认（课前2分钟）

- [ ] 所有检查项完成
- [ ] 教案已打开或打印，放在手边可随时查看
- [ ] 幻觉演示案例已测试，确认可用的案例编号已记录
- [ ] 手机设为静音
- [ ] 确认上课时间，准备开始

### 三问法应用于AI（幻觉环节使用，投影展示）

```
┌────────────────────────────────────────────────┐
│        信息判断三问 → 应用于AI                    │
│                                                │
│  ① 谁在说？                                     │
│    → AI在说。它是一个语言模型——                   │
│      工作方式是"预测下一个最可能出现的字"。        │
│      它不是律师、不是医生、不是政府工作人员。      │
│      它见过大量文字，知道法律条文"长什么样子"，    │
│      所以编出来的东西看起来很像真的。              │
│                                                │
│  ② 能不能验证？                                  │
│    → 能。                                        │
│      用搜索引擎搜法律原文，一对比就知道对不对。    │
│      凡是具体数字、法律条文、地址电话——           │
│      搜索一下就能验证。                           │
│                                                │
│  ③ 动机是什么？                                  │
│    → AI不想骗你。它没有恶意。                     │
│      但AI被设计成"尽量给你一个回答"——             │
│      它宁可编一个看起来对的答案，                  │
│      也不会说"对不起我不知道"。                   │
│      它的"动机"是讨好你、让你满意。               │
└────────────────────────────────────────────────┘
```

## 备用方案

### 备用方案A：个别学员电脑无法联网

**现象**：1-2台电脑连不上网络，其他电脑正常。

**处理步骤**：

1. 先尝试基本排查：
   - 检查Wi-Fi是否已连接（看任务栏网络图标）
   - 断开Wi-Fi重新连接
   - 重启浏览器
2. 如果3分钟内无法解决：
   - 安排该学员与旁边的同学**共用一台电脑**
   - 两人一起操作，一个问AI一个记录，互相讨论效果可能更好
3. 课后再排查这台电脑的问题

### 备用方案B：AI工具无法访问

**现象**：电脑联网正常，但AI工具网页打不开或加载异常。

**处理步骤**：

1. 先尝试基本排查：
   - 刷新页面
   - 清除浏览器缓存，重新打开
   - 换一个浏览器试试（Chrome / Edge / Firefox）
2. 如果是个别电脑问题：
   - 安排该学员与旁边同学共用
3. 如果是全场都打不开：
   - 切换到备用AI工具（提前准备2-3个可用的AI工具网址，如豆包、通义千问、Kimi等国内可直接访问的工具）
   - 如果所有AI工具都不可用：改为**讲师演示为主**模式——讲师用手机热点连接自己电脑演示，学员观看并口头参与讨论
   - 练习环节改为：讲师读出学员口述的提问，输入AI，大家一起看结果并讨论

### 备用方案C：AI响应缓慢

**现象**：AI能访问，但回复速度很慢（超过30秒才开始生成）。

**处理步骤**：

1. 减少同时使用AI的设备数量：
   - 演示环节只用讲师电脑
   - 练习环节分批进行（先一半学员操作，另一半观察，然后换）
2. 如果持续缓慢：
   - 切换到响应更快的备用AI工具
   - 缩短练习任务的轮次要求（任务2的3轮改为2轮）

### 备用方案D：投影仪/大屏幕故障

**现象**：投影仪不亮，或者连接不上讲师电脑。

**处理步骤**：

1. 检查连接线是否插紧，切换输入源
2. 如果3分钟内无法解决：
   - 让学员围过来看讲师电脑屏幕（适用于5人以内小班）
   - 或者讲师走到每个学员旁边，在学员电脑上演示关键步骤
3. 提问对比卡已经打印发给了每个人，即使没有投影也能参考

### 备用方案E：学员人数超出预期

**现象**：来了超过10个人，电脑不够。

**处理步骤**：

1. 两人一台电脑，**一人操作一人观察，中间换人**
2. 练习环节时间适当延长，确保每人都有动手机会
3. 如果超过15人，考虑分两批上课（与组织者沟通）

## 讲师备忘

- 这是学员**第一次系统使用**AI工具，但**不是**第一次接触——第1节课他们已经用AI验证过仓库管理员工资和社保信息。开场时可以引用这个经验："上节课大家已经体验过了。"
- 幻觉演示是本节课的重点之一，效果好坏直接影响学员对AI的正确认知。**务必课前测试案例**，不要在课堂上才发现案例失效。
- **三问法回调是有机整合的关键连接点**。幻觉演示之后，要显式回调第1节的"谁在说、真不真、为什么"，让学员发现同一套思维工具换了场景照样能用。讲到AI的"动机"（讨好你）时，跟第1节分析假招聘帖的"动机"（骗你钱）做对比，学员会印象深刻。
- 提问对比卡是学员带走的实物参考，帮助他们课后自己练习时有据可依。
- 如果有学员上节课缺席（没学过搜索），在幻觉验证环节可以顺便简单教他们用搜索引擎验证——这也是复习第1节内容的机会。
- 隐私保护要反复强调：演示和练习中都用化名，不输入真实个人敏感信息。
- **务必在课后小结时指定第3节的回顾人选**。第3节起正式启用"学员主讲回顾"机制，如果这里忘了指定，第3节开头会尴尬。选一个表现相对积极的学员作为第一个回顾者。
