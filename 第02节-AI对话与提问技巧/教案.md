# 第2节：AI对话与提问技巧 —— 剧本式教案

## 课时信息

| 项目 | 内容 |
|:---|:---|
| 所属阶段 | 基础课（第2节/共14节） |
| 主讲 | 梅杭俊 |
| 时长 | 45 分钟 |
| 班级规模 | 5-10 人 |
| 前置要求 | 第1节-搜索与信息素养 |

## 教学目标

学完本节课，学员能够：
1. 使用网页版AI进行基本对话
2. 用"说清身份 + 说清需求 + 说清限制"的框架提出有效问题
3. 进行多轮对话，追问和修正AI的回答
4. 识别AI幻觉现象，用第1节学的"信息判断三问"评估AI回答的可信度
5. 建立"信但验"的AI使用习惯——AI给方向，搜索做验证

### 设计说明

> 本节课是学员系统学习AI对话的第一课，但**不是**他们第一次接触AI——第1节课中，AI已经作为信息验证渠道之一自然出场（用AI验证仓库管理员工资水平、验证社保信息）。因此本节课的定位是"深入学AI"而非"认识AI"。
>
> 核心教学设计有两个重点：
> 1. **提问三要素**（说清身份+需求+限制）：这是AI对话的核心技能，通过模糊vs精确对比演示建立直觉，通过多轮对话演示建立"AI是可以反复沟通的助手"的认知。
> 2. **AI幻觉与三问法回调**：在演示AI幻觉之后，显式回调第1节建立的"信息判断三问"框架，让学员发现同一套思维工具换了场景照样能用。这强化了三问法作为底层能力的定位，也让学员理解AI的特殊"动机"——它不是想骗你，但它为了讨好你宁可编一个答案。
>
> 本节课为第3节（办公软件与AI协作）做直接铺垫：第3节会让学员用AI生成文字内容然后自己排版、用AI写Excel公式然后自己验证——这要求学员既会跟AI对话，也知道AI不能盲信。

## 开场回顾 + 今日目标（2分钟）

### 讲师说

> "上节课我们学了怎么找信息——去哪找、怎么搜、怎么判断靠不靠谱。大家还记得信息判断的三个问题吗？——谁在说、真不真、为什么。上节课练习的时候，大家还用了一下AI来验证社保信息，有印象吧？"

> "今天我们就来深入学AI——上节课只是体验了一下，今天要系统地学怎么跟它对话。**搜索和AI是一对组合拳**——AI帮你快速得到答案，搜索帮你验证答案对不对。"

> "今天的目标就一句话：**学会跟AI说话，让它给你有用的回答，同时别被它骗了。**"

### 动作

- 不用投影，直接跟学员对话
- 提到"三个问题"的时候可以等一下，看有没有学员能说出来（谁在说、真不真、为什么）
- 如果有学员答出来，肯定他："对，记得很清楚"

## 一、AI是什么——定位与边界（5分钟）

### 讲师说

> "在正式用AI之前，我们先搞清楚它是什么。AI不是神，也不是废物，它是一个**很会说话的助手**——记住这个定位。"

> "为什么说'很会说话'？因为AI的工作方式，本质上就是**预测下一个最可能出现的字**。你给它一句话，它根据自己见过的大量文字材料，猜后面应该接什么字、什么句子。它见过法律文件，所以它能写出看起来像法律条文的东西——但它没有真的翻过法典。它见过医学论文，所以它能写出看起来像医学建议的东西——但它不是医生。"

> "这就解释了一个很重要的事：**AI写的东西看起来很像真的，但不保证是真的。** 就像一个什么都敢说的同事——说得很自信，但你不能不验证就信他。"

### 投影展示

在投影上展示以下对比表格：

| AI擅长的 | AI做不到的 |
|:---|:---|
| 回答各种问题，给你思路 | 替你做决定 |
| 帮你写文章、写请假条、写代码 | 保证100%正确 |
| 翻译文字 | 访问你的私人账户 |
| 分析和总结大量信息 | 知道最新的新闻（有滞后） |
| 扮演各种角色（面试官、老师等） | 替代你的判断力 |

### 讲师说

> "左边这些事情，AI做得又快又好——后面的课我们都会用到。右边这些事情，它做不到。特别注意'不保证100%正确'和'不能替代你的判断力'这两条。所以我们的原则是三个字：**信但验**——先信AI给的方向，再用搜索去验证关键信息。"

### 演示：打开AI工具（1分钟）

**操作步骤：**

1. 打开浏览器，地址栏输入AI工具的网址
   - 【讲师注：根据实际选用的工具填写地址。如用Claude网页版，输入 `claude.ai`；如用国内工具，输入对应地址】
2. 登录账号（如果学员有统一账号，提前写在黑板上）
3. 找到对话输入框

### 讲师说

> "大家看，这个界面很简单——下面有个输入框，你在里面打字，按回车或者点发送，AI就会回答你。我们来试一下。"

**演示输入：**
```
你好，请用一句话介绍你自己。
```

> "看，它回答了。就这么简单，跟聊天一样。但是——怎么聊、怎么问，结果差别非常大。这就是接下来要讲的重点。"

### 学员屏幕

学员此时不需要操作，看投影即可。（如果学员已经迫不及待打开了AI工具，不用制止，让他们自己先试着聊两句也可以。）

### 可能的卡点

| 现象 | 原因 | 讲师应对 |
|:---|:---|:---|
| 学员问"AI怎么知道这么多" | 对AI原理好奇 | "它读过互联网上几十亿页的文字，所以什么话题都能聊。但'读过'不等于'理解'，后面讲幻觉你就明白了" |
| 学员问"这个跟百度有什么区别" | 第1节用过搜索，现在想区分 | "百度给你一堆链接让你自己看，AI直接给你一个消化过的答案。各有优缺点——AI快但可能错，搜索慢但你能看到原始出处。两个配合用最好" |

## 二、提问的艺术（15分钟）

### 第一部分：模糊 vs 精确提问对比演示（7分钟）

#### 讲师说

> "我先演示一个东西，大家看完就明白了。上节课我们学了怎么找工作——今天的AI练习也围绕工作中会遇到的事情。我现在要让AI帮我写一封请假条。我先用一种'懒'的方式问："

**演示——模糊提问：**

在AI对话框中输入：
```
帮我写个请假条
```

等AI回答完后，**不要关闭这个对话**。

#### 讲师说

> "大家看看这个结果——它确实写了，但是写得很笼统，请假原因是'个人原因'，也不知道写给谁，格式也一般。这是因为你没告诉它细节，它只能瞎猜。"

> "现在我换一种方式问，大家注意看区别："

**开一个新对话。** 演示——精确提问：

```
我是一名工厂流水线工人，需要写一封请假条给车间主管王强。请假原因是回老家办身份证，需要请假3天（3月10日到3月12日）。请假条要正式、简洁，200字以内。
```

等AI回答完。

#### 讲师说

> "大家对比一下这两个结果。第二个是不是好太多了？原因、时间、收件人、语气全都对了。**同一个AI，问法不同，结果天差地别。**"

> "所以，问AI不是随便打两个字就行，你得**把话说清楚**。怎么说清楚？三个要素："

#### 学员屏幕

学员此时看投影，不需要操作。

#### 可能的卡点

| 现象 | 原因 | 讲师应对 |
|:---|:---|:---|
| AI对模糊提问也写得还不错 | AI越来越聪明，有时候模糊提问结果也可以 | "确实写了，但你看它猜的请假原因对不对？收件人猜对了吗？你越不给信息，它猜错的概率越大" |
| 学员想自己试 | 积极性高 | "先看我演示完，等下练习环节你们自己动手" |

### 第二部分：提问三要素（5分钟）

#### 投影展示

```
提问三要素：
┌─────────────┐
│ 1. 说清你是谁 │  → "我是工厂工人" / "我是外卖骑手"
│ 2. 说清要什么 │  → "帮我写请假条" / "帮我算加班费"
│ 3. 说清限制   │  → "200字以内" / "用表格形式" / "语气正式"
└─────────────┘
```

#### 讲师说

> "第一，**说清你是谁**。你告诉AI你是工厂工人，它回答的内容就会贴合工厂的场景；你不说，它可能给你写一封办公室白领风格的请假条。"

> "第二，**说清要什么**。不要说'帮我写个东西'，要说清楚是请假条、简历、还是投诉信。"

> "第三，**说清限制**。比如字数、格式、语气、用途。限制越清楚，AI越不容易跑偏。"

> "我给大家发一张'提问对比卡'，上面有几组例子，大家可以对着看。"

**此时分发《提问对比卡》。**

#### 动作

- 分发提问对比卡（提前打印好，每人一张）
- 给学员30秒看一下对比卡上的例子
- 提问对比卡可以带走，课后自己练习时参考

#### 学员屏幕

学员此时不需要操作，看投影和手中的对比卡。

### 第三部分：多轮对话技巧（3分钟）

#### 讲师说

> "还有一个重要技巧——**多轮对话**。AI不是用一次就完事，你可以跟它来回聊，就像跟一个助手反复沟通。"

**演示：继续刚才精确提问的请假条对话。** 在同一对话中输入：

```
写得不错，但有两个修改：1. 把"恳请批准"改成"望批准为盼" 2. 落款加上我的工号：GD-20250316
```

等AI回答。

#### 讲师说

> "看到了吗？它记住了前面的内容，直接在原来的基础上改。这就是多轮对话的好处——你不用每次都重新说一遍所有信息。"

> "再演示一个更实用的——**角色设定**。"

**在新对话中输入：**
```
你现在扮演一个工厂的HR面试官。我要面试一个电子厂操作员的岗位。请你问我3个常见的面试问题，一次问一个，等我回答后再问下一个，最后给我打分和改进建议。
```

#### 讲师说

> "这种方式特别适合面试练习。你让AI扮演面试官，它就会一个一个问你问题——不用花钱，不用麻烦别人，随时随地可以练。"

> "好，让AI问第一个问题……"

**让AI回出第一个问题即可，不需要完整走完面试流程。**

> "这个大家练习时间可以自己试。接下来我们讲一个非常重要的事情——AI会犯错。"

#### 学员屏幕

学员看投影。如果有学员跟着在自己电脑上试了多轮对话，不用制止。

#### 可能的卡点

| 现象 | 原因 | 讲师应对 |
|:---|:---|:---|
| 学员在新对话里追问，AI说"不知道你说的是什么" | 开了新对话，AI没有上下文 | "注意——追问要在**同一个对话**里。你开了新对话，AI就忘了前面的内容。就像跟一个新人说话，你得重新说一遍" |
| 角色设定后AI没有按要求一次问一个 | AI有时会一次给出所有问题 | "没关系，你可以再追一句：'请一次只问一个问题，等我回答后再问下一个'" |

## 三、AI幻觉与三问法（10分钟）

> **设计意图**：本环节有两个目的。第一，让学员亲眼看到AI犯错，建立"不能盲信"的认知。第二，显式回调第1节的"信息判断三问"框架，让学员发现同一套思维工具对AI同样适用——这强化了三问法作为底层能力的定位。学员在这里建立的"信但验"习惯，会直接用于第3节（检查AI生成的简历内容）和第5节以后（验证AI生成的代码）。

### 讲师说

> "前面我说了，AI不保证100%正确。现在我给大家**亲眼看看**它是怎么犯错的。"

### 幻觉演示案例一：虚构法律条文（主演示，4分钟）

**在新对话中输入：**
```
请详细解释《中华人民共和国劳动法》第九十三条的具体内容和适用场景。
```

【讲师注：中国《劳动法》一共只有107条。但如果问一个具体条数（特别是较高的条数），AI有较大概率会编造一个不存在的条文内容，而且说得非常像真的。**课前务必测试此案例**，确认AI确实会编造。如果AI回答正确（说此条不存在或正确引用），请改用备用案例。】

等AI回答完。AI大概率会"自信地"编出一段法律条文。

#### 讲师说

> "大家看，它回答得很认真对吧？有条文内容，还有适用场景分析。**但这是假的。** 我们来验证一下。"

**打开搜索引擎，搜索：**
```
中华人民共和国劳动法 全文
```

> "大家看，《劳动法》一共只有107条。第93条的真实内容是……和AI说的完全不一样。（或：根本就没有这一条。）"

> "AI就是这样——它**不知道自己不知道**。它不会说'对不起我不确定'，它会非常自信地编一个答案出来。这种现象叫做**AI幻觉**。"

### 幻觉演示案例二（备用，如案例一失败则用此案例）

**在新对话中输入：**
```
请介绍一下成都市金牛区劳动保障服务中心的地址、电话和办公时间。
```

【讲师注：AI大概率会编造一个看似合理但实际错误的地址和电话。课前测试确认。】

等AI回答完。

> "它给了地址和电话——我们搜索验证一下。"

**搜索验证。** 指出AI给出的具体信息（地址、电话）与实际不符。

#### 学员屏幕

学员看投影。这个环节是讲师单独演示，学员不需要操作。

#### 可能的卡点

| 现象 | 原因 | 讲师应对 |
|:---|:---|:---|
| AI没有幻觉，给了正确回答 | 模型更新后可能修复了某些问题 | 切换到备用案例。如果备用案例也不幻觉，用第三个：问AI一个虚构的成都本地餐馆名称和地址，然后搜索验证不存在 |
| 学员说"那AI不能用了？" | 看到幻觉后过度恐惧 | "不是不能用，是不能盲信。它写文章、给思路、帮你算东西，这些做得很好。只是涉及具体事实——法律条文、地址、数字——你要验证一下。就像你的同事给你出主意，主意不错但你得自己确认细节" |

### 三问法回调：用第1节的框架分析AI（4分钟）

#### 讲师说

> "AI编了一个法律条文出来，说得很像真的。大家还记得上节课的三个问题吗？——**谁在说、真不真、为什么**。上节课我们用这三个问题分析了一个假招聘帖。今天我们对AI也来问一遍。"

（投影展示）

#### 投影内容——三问法应用于AI

```
信息判断三问 → 应用于AI：

① 谁在说？
   → AI在说。
   AI是一个语言模型——它的工作方式是"预测下一个最可能出现的字"。
   它不是律师、不是医生、不是政府工作人员。
   它见过大量文字，知道法律条文"长什么样子"，
   所以它编出来的东西看起来很像真的——但它没有真的翻过法典。

② 能不能验证？
   → 能。
   用搜索引擎搜法律原文，一对比就知道AI说的对不对。
   凡是具体数字、法律条文、地址电话——搜索一下就能验证。

③ 动机是什么？
   → AI不想骗你。它没有恶意。
   但AI被设计成"尽量给你一个回答"——
   它宁可编一个看起来对的答案，也不会说"对不起我不知道"。
   它的"动机"是讨好你、让你满意。
```

#### 讲师说

> "第一个问题，**谁在说**？AI在说。AI是什么？它是一个语言模型——上课开头我说过，它的工作方式就是预测下一个最可能出现的字。它不是律师，没有翻过法典。它只是见过大量的文字，知道法律条文大概'长什么样'，所以它编出来的东西看起来很像真的。"

> "第二个问题，**能不能验证**？能。刚才我们用搜索引擎搜到了劳动法全文，一对比就发现AI编的。凡是涉及具体数字、法律条文、地址电话这些事实性内容，搜索一下就能验证。这就是上节课学的搜索能力——它就是你验证AI的武器。"

> "第三个问题，**动机是什么**？这个跟上节课分析招聘帖不一样。发假招聘信息的人是想骗你的钱。AI不想骗你——它没有恶意。但AI有一个特殊的'动机'：它被设计成**尽量给你一个回答**。它宁可编一个看起来对的答案，也不会说'对不起我不确定'。用大白话说——**它不骗你，但它为了讨好你会编东西**。就像一个很热心但不太靠谱的同事，你问他什么他都敢答。"

> "所以结论是什么？信但验。AI给你方向，搜索帮你验证。**上节课的三个问题，对招聘信息管用，对AI一样管用，以后对任何信息来源都管用。** 这不是三个死记硬背的问题，而是一个思维习惯。"

#### 投影展示——信但验原则

```
信但验：
  AI → 给方向、给初稿、给思路
  搜索 → 验证事实、核实数据
  你的判断 → 做最终决定

记住：AI是你的助手，不是你的老师。
      搜索是你的证人，不是你的敌人。
      两个一起用，才靠谱。
```

#### 讲师说

> "最后记住一句话：**AI是你的助手，不是你的老师；搜索是你的证人，不是你的敌人。两个一起用，才靠谱。**"

## 四、练习（11分钟）

### 讲师说

> "好，现在大家动手练。三个任务，按顺序做，每个任务完成后举手跟我确认。先完成的同学帮旁边还没完成的同学。"

### 任务1：对比练习（3分钟）

**讲师投影展示任务：**

```
任务1：感受模糊和精确提问的差距

1. 开一个新对话，输入：帮我写个辞职信
2. 再开一个新对话，输入：
   我是电子厂的流水线操作员，在这里工作了2年。
   我想写一封辞职信给车间主管李伟，辞职原因是打算回老家发展。
   要求语气礼貌但坚定，提到按劳动法提前30天通知，300字以内。
3. 对比两个回答，想一想：哪个更能直接用？差在哪里？

完成标准：能用自己的话说出两个回答的差别。
```

#### 可能的卡点

| 现象 | 原因 | 讲师应对 |
|:---|:---|:---|
| 学员不知道怎么开新对话 | 第一次用AI工具 | 走过去指出来：界面上通常有"新对话"或"+"按钮 |
| 学员分不清两个回答的差别 | 可能两个回答看起来都"还行" | 引导他看具体细节："收件人对不对？原因写了吗？语气像你想要的吗？" |

### 任务2：多轮对话——简历优化（5分钟）

**讲师投影展示任务：**

```
任务2：用多轮对话优化简历

1. 开一个新对话，输入（可以用化名，保护隐私）：
   我叫张伟（化名），男，25岁，初中毕业。
   做过2年电子厂操作员，1年外卖骑手。
   现在想找一份仓库管理员的工作。请帮我写一份简历。
2. 看AI写的初稿，找出你觉得不好的地方
3. 在同一个对话中跟AI说要改什么，比如：
   - "工作经历部分太简单了，帮我加上具体做了什么"
   - "技能部分帮我加上'熟悉仓库管理系统WMS'"
   - "把简历针对这个招聘要求调整一下：[贴一段招聘要求]"
4. 至少来回对话3轮

完成标准：简历经过3轮修改，比初稿明显更好。
隐私提示：请使用化名，不要输入真实的身份证号、手机号等个人信息。
```

#### 讲师注意

> 这个任务延续了第1节"找仓库管理员工作"的贯穿场景，让学员体验用AI辅助求职的完整链路：第1节搜索职位和判断信息真假 → 第2节用AI写简历并反复优化 → 第3节将AI写的内容在Word中排版成正式简历。

#### 可能的卡点

| 现象 | 原因 | 讲师应对 |
|:---|:---|:---|
| 学员不知道怎么"找不好的地方" | 对简历没有经验 | "看看有没有这些问题：工作经历有没有写具体做了什么？有没有编造你没有的技能？语气是不是太夸张了？" |
| 学员追问时开了新对话 | 不理解"同一个对话"的含义 | "在同一个聊天窗口里接着打字就行，不要点'新对话'。这样AI能记住前面写的内容" |
| AI编造了学员没有的经历或证书 | AI幻觉在简历场景中的典型表现 | "看到了吗？AI给你编了一段你没有的经历。**这就是幻觉**——刚才讲的三个问题又派上用场了。AI写的简历你一定要检查，假的东西删掉" |

### 任务3：验证练习（3分钟）

**讲师投影展示任务：**

```
任务3：验证AI的回答

1. 在AI对话中问：成都市2026年的最低工资标准是多少？
2. 记下AI的回答
3. 打开搜索引擎，搜索验证这个数字是否正确
   （提示：试试 "成都 2026 最低工资标准 site:gov.cn"）
4. 如果不对，找到正确数字

完成标准：
能说出AI的回答对不对，如果不对，正确的数字是多少，
你是在哪个网站找到的。

想一想：对AI的回答，"谁在说"？它说的数字"能验证"吗？
```

#### 可能的卡点

| 现象 | 原因 | 讲师应对 |
|:---|:---|:---|
| 学员找不到政府网站的数据 | 搜索技巧不熟练 | "用上节课学的 site:gov.cn 试试。或者直接搜 '成都 2026 最低工资标准'" |
| AI给的数字碰巧是对的 | 部分常识性数据AI可能知道 | "对的也好，关键是你**验证了**。如果你不搜，你怎么知道它是对的？养成验证的习惯" |
| 学员直接相信AI没去搜 | 觉得AI已经说了就是对的 | "你怎么知道它说的是真的？上课才演示了AI编法律条文——同一个AI，刚才还在编东西呢。去搜一下" |

### 巡场要点

- **任务1**：重点看学员能否说出差距原因（"因为第二个问题给了身份、原因、字数这些信息"）
- **任务2**：重点看学员是否在**同一个对话**中追问（不是开新对话），是否真的来回了3轮。如果AI编造了学员没有的经历，这是绝佳的教学时机——走过去说"看，这就是幻觉"
- **任务3**：重点看学员是否真的去搜了，而不是直接相信AI的回答
- **先完成的学员**：引导去帮助未完成的同学——"你做完了？去看看旁边XX做到哪了，帮他一下"
- **如果大部分人卡在同一个地方**：暂停练习，统一在投影上演示那一步

## 课后小结（2分钟）

### 讲师说

> "好，时间到了。今天学了三件事：
>
> **第一，跟AI对话的方法。** 三个要素：说清你是谁、说清要什么、说清限制。多轮对话——像跟助手反复沟通一样，追问、修正、让AI改。
>
> **第二，AI会犯错——幻觉。** AI是一个很热心但不保证靠谱的助手。它不是想骗你，但它为了给你一个回答宁可编。
>
> **第三，上节课的三个问题对AI一样管用。** 谁在说——AI，一个语言模型。能验证吗——能，用搜索。动机——不是恶意，但会讨好你。**信但验，这是使用AI的底线。**
>
> 下节课我们学Word和Excel，到时候就把AI用起来——让AI帮你写内容，你自己排版和验证。今天学的提问技巧和验证习惯，下节课马上就用得到。"

### 指定下节课回顾人选

> "最后一件事——从下节课开始，每节课开头都会请一位同学来回顾上节课的内容。下节课请XXX来讲，好吗？不用紧张，用自己的话说就行，两三分钟。你今天学了什么就说什么。"

【讲师注：这是为第3节的"同伴教学"机制做准备。第3节起正式启用学员主讲回顾。务必在这里指定人选，否则第3节开头会没人准备。选一个表现相对积极、不太怕开口的学员作为第一个回顾者。】

## FAQ

### Q1："AI工具要花钱吗？"

> **讲师回答**："网页版有免费额度，日常使用够了。后面我们学到Claude Code的时候再讲付费的事情。现在不用担心费用问题。"

### Q2："AI会不会泄露我的隐私？"

> **讲师回答**："这个问题问得好。原则是：**不要在AI对话中输入你的身份证号、银行卡号、密码这些敏感信息**。像刚才的简历练习，我让大家用化名，就是这个原因。AI是工具，跟在公共电脑上打字一样，不要输入你不愿意被别人看到的东西。"

### Q3："AI回答的内容我可以直接用吗？比如直接交给领导？"

> **讲师回答**："可以用，但要先检查。特别是涉及数字、日期、法律条文、公司名称这些**事实性内容**，一定要验证。文字表达、格式这些可以直接用。养成一个习惯：用AI生成，用搜索验证，用自己的判断做最终决定。"

### Q4："AI说的东西跟搜索结果不一样，听谁的？"

> **讲师回答**："听证据的。哪边有官方来源（政府网站、法律原文、官方公告），就信哪边。如果都没有明确官方来源，那就多找几个来源交叉对比。还记得上节课的三个问题吗？——看看每个渠道的'谁在说'和'动机是什么'，判断哪个更可信。"

### Q5："我打字慢，跟AI对话效率很低怎么办？"

> **讲师回答**："没关系，你可以先在记事本里把要问的话打好，然后复制粘贴到AI对话框里。而且AI不着急，你慢慢打它也会等你。另外，打字速度会越用越快，这本身就是在练。"
