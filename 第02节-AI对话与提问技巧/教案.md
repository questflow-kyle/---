## 教学目标

学完本节课，学员能够：
1. 使用网页版AI进行基本对话
2. 用"说清身份 + 说清需求 + 说清限制"的框架提出有效问题
3. 进行多轮对话，追问和修正AI的回答
4. 识别AI幻觉现象，建立交叉验证习惯

## 开场回顾 + 今日目标（2分钟）

### 讲师说

> "上节课我们学了搜索——怎么用关键词快速找到靠谱信息，怎么交叉验证。今天我们学一个新工具：AI对话。学完之后你会发现，**搜索和AI是一对组合拳**——AI帮你快速得到答案，搜索帮你验证答案对不对。"

> "今天的目标就一句话：**学会跟AI说话，让它给你有用的回答，同时别被它骗了。**"

---

## 一、AI能做什么、不能做什么（5分钟）

### 讲师说

> "在正式用AI之前，我们先搞清楚它是什么。AI不是神，也不是废物，它是一个**很会说话的助手**——记住这个定位。"

### 板书/投影展示

在投影上展示以下对比表格：

| AI能做的 | AI不能做的 |
|:---|:---|
| 回答各种问题 | 替你做决定 |
| 帮你写文章、写请假条 | 保证100%正确 |
| 翻译文字 | 访问你的私人账户 |
| 分析和总结信息 | 知道最新的新闻（有滞后） |
| 写代码（后面会学到） | 替代你的判断力 |

### 讲师说

> "左边这些事情，AI做得又快又好。右边这些事情，它做不到。特别注意第二条——**AI不保证100%正确**。它有时候会非常自信地跟你说一个完全错误的东西，就像一个什么都敢说的同事。所以我们的原则是四个字：**信但验**——先相信它给的方向，再用搜索去验证。"

### 演示：打开AI工具（1分钟）

**操作步骤：**

1. 打开浏览器，地址栏输入AI工具的网址
   - 【讲师注：根据实际选用的工具填写地址。如用Claude网页版，输入 `claude.ai`】
2. 登录账号（如果学员有统一账号，提前写在黑板上）
3. 找到对话输入框

### 讲师说

> "大家看，这个界面很简单——下面有个输入框，你在里面打字，按回车，AI就会回答你。我们来试一下。"

**演示输入：**
```
你好，请用一句话介绍你自己。
```

> "看，它回答了。就这么简单，跟聊天一样。但是——怎么聊、怎么问，结果差别非常大。这就是接下来要讲的重点。"

## 二、提问的艺术（15分钟）

### 第一部分：模糊 vs 精确提问对比演示（7分钟）

### 讲师说

> "我先演示一个东西，大家看完就明白了。我现在要让AI帮我写一封请假条。我先用一种'懒'的方式问：**"

**演示——模糊提问：**

在AI对话框中输入：
```
帮我写个请假条
```

等AI回答完后，**不要关闭这个对话**。

### 讲师说

> "大家看看这个结果——它确实写了，但是写得很笼统，请假原因是'个人原因'，也不知道写给谁，格式也一般。这是因为你没告诉它细节，它只能瞎猜。"

> "现在我换一种方式问，大家注意看区别：**"

**开一个新对话。** 演示——精确提问：

```
我是一名工厂流水线工人，需要写一封请假条给车间主管王强。请假原因是回老家办身份证，需要请假3天（3月10日到3月12日）。请假条要正式、简洁，200字以内。
```

等AI回答完。

### 讲师说

> "大家对比一下这两个结果。第二个是不是好太多了？原因、时间、收件人、语气全都对了。**同一个AI，问法不同，结果天差地别。**"

> "所以，问AI不是随便打两个字就行，你得**把话说清楚**。怎么说清楚？三个要素：**"

### 第二部分：提问三要素（5分钟）

### 板书/投影

```
提问三要素：
┌─────────────┐
│ 1. 说清你是谁 │  → "我是工厂工人" / "我是外卖骑手"
│ 2. 说清要什么 │  → "帮我写请假条" / "帮我算加班费"
│ 3. 说清限制   │  → "200字以内" / "用表格形式" / "语气正式"
└─────────────┘
```

### 讲师说

> "第一，**说清你是谁**。你告诉AI你是工厂工人，它回答的内容就会贴合工厂的场景；你不说，它可能给你写一封办公室白领风格的请假条。"

> "第二，**说清要什么**。不要说'帮我写个东西'，要说清楚是请假条、简历、还是投诉信。"

> "第三，**说清限制**。比如字数、格式、语气、用途。限制越清楚，AI越不容易跑偏。"

> "我给大家发一张'提问对比卡'，上面有几组例子，大家可以对着看。"

**此时分发《提问对比卡》。**

### 第三部分：多轮对话技巧（3分钟）

### 讲师说

> "还有一个重要技巧——**多轮对话**。AI不是用一次就完事，你可以跟它来回聊，就像跟一个助手反复沟通。"

**演示：继续刚才精确提问的请假条对话。** 在同一对话中输入：

```
写得不错，但有两个修改：1. 把"恳请批准"改成"望批准为盼" 2. 落款加上我的工号：GD-20250316
```

等AI回答。

### 讲师说

> "看到了吗？它记住了前面的内容，直接在原来的基础上改。这就是多轮对话的好处——你不用每次都重新说一遍所有信息。"

> "再演示一个更实用的——**角色设定**。"

**在新对话中输入：**
```
你现在扮演一个工厂的HR面试官。我要面试一个电子厂操作员的岗位。请你问我3个常见的面试问题，一次问一个，等我回答后再问下一个，最后给我打分和改进建议。
```

### 讲师说

> "这种方式特别适合面试练习。你让AI扮演面试官，它就会一个一个问你问题——不用花钱，不用麻烦别人，随时随地可以练。"

> "好，让AI问第一个问题……"

**让AI回出第一个问题即可，不需要完整走完面试流程。**

> "这个大家练习时间可以自己试。接下来我们讲一个非常重要的事情——AI会犯错。"

---

## 三、AI幻觉演示（10分钟）

### 讲师说

> "前面我说了，AI不保证100%正确。现在我给大家**亲眼看看**它是怎么犯错的。"

### 幻觉演示案例一：虚构法律条文（主演示，5分钟）

**在新对话中输入：**
```
请详细解释《中华人民共和国劳动法》第九十三条的具体内容和适用场景。
```

【讲师注：中国《劳动法》一共只有107条。但如果问一个具体条数（特别是较高的条数），AI有较大概率会编造一个不存在的条文内容，而且说得非常像真的。**课前务必测试此案例**，确认AI确实会编造。如果AI回答正确（说此条不存在或正确引用），请改用备用案例。】

等AI回答完。AI大概率会"自信地"编出一段法律条文。

### 讲师说

> "大家看，它回答得很认真对吧？有条文内容，还有适用场景分析。**但这是假的。** 我们来验证一下。"

**打开搜索引擎，搜索：**
```
中华人民共和国劳动法 全文
```

> "大家看，《劳动法》一共只有107条。第93条的真实内容是……和AI说的完全不一样。（或：根本就没有这一条。）"

> "AI就是这样——它**不知道自己不知道**。它不会说'对不起我不确定'，它会非常自信地编一个答案出来。这种现象叫做**AI幻觉**。"

### 幻觉演示案例二（备用，如案例一失败则用此案例，3分钟）

**在新对话中输入：**
```
请介绍一下成都市金牛区劳动保障服务中心的地址、电话和办公时间。
```

【讲师注：AI大概率会编造一个看似合理但实际错误的地址和电话。课前测试确认。】

等AI回答完。

> "它给了地址和电话——我们搜索验证一下。"

**搜索验证。** 指出AI给出的具体信息（地址、电话）与实际不符。

### 关键原则讲解（2分钟）

### 板书/投影

```
信但验：
  AI → 给方向
  搜索 → 找证据
  交叉验证 → 出结论
```

### 讲师说

> "所以我们的原则就是三个字：**信但验**。"
>
> "**信**——AI给你的回答，大部分时候是有用的，可以作为起点。"
>
> "**验**——但涉及到事实、数字、法律、地址这些东西，一定要用搜索去验证。"
>
> "上节课学的搜索能力，就是你验证AI的武器。搜索和AI不是二选一，是**组合使用**的。"

> "记住一句话：**AI是你的助手，不是你的老师；搜索是你的证人，不是你的敌人。两个一起用，才靠谱。**"

## 四、练习（13分钟）

### 讲师说

> "好，现在大家动手练。三个任务，按顺序做，每个任务完成后举手跟我确认。先完成的同学帮旁边的同学。"

### 任务1：对比练习（4分钟）

**讲师投影展示任务：**

> **任务1：感受模糊和精确提问的差距**
>
> 1. 开一个新对话，输入：`帮我写个辞职信`
> 2. 再开一个新对话，输入：`我是电子厂的流水线操作员，在这里工作了2年。我想写一封辞职信给车间主管李伟，辞职原因是打算回老家发展。要求语气礼貌但坚定，提到按劳动法提前30天通知，300字以内。`
> 3. 对比两个回答，想一想：哪个更能直接用？差在哪里？
>
> **完成标准：** 能用自己的话说出两个回答的差别。

### 任务2：多轮对话——简历优化（5分钟）

**讲师投影展示任务：**

> **任务2：用多轮对话优化简历**
>
> 1. 开一个新对话，输入（可以用化名，保护隐私）：
>    ```
>    我叫张伟（化名），男，25岁，初中毕业。做过2年电子厂操作员，1年外卖骑手。现在想找一份仓库管理员的工作。请帮我写一份简历。
>    ```
> 2. 看AI写的初稿，找出你觉得不好的地方
> 3. 在同一个对话中跟AI说要改什么，比如：
>    - "工作经历部分太简单了，帮我加上具体做了什么"
>    - "技能部分帮我加上'熟悉仓库管理系统WMS'"
>    - "把简历针对这个招聘要求调整一下：[贴一段招聘要求]"
> 4. 至少来回对话 **3轮**
>
> **完成标准：** 简历经过3轮修改，比初稿明显更好。
>
> **隐私提示：** 请使用化名，不要输入真实的身份证号、手机号等个人信息。

### 任务3：验证练习（4分钟）

**讲师投影展示任务：**

> **任务3：验证AI的回答**
>
> 1. 在AI对话中问：`成都市2026年的最低工资标准是多少？`
> 2. 记下AI的回答
> 3. 打开搜索引擎，搜索验证这个数字是否正确
> 4. 如果不对，找到正确数字
>
> **完成标准：** 能说出AI的回答对不对，如果不对，正确的数字是多少，你是在哪个网站找到的。

### 巡场要点

- 任务1：重点看学员能否说出差距原因（"因为第二个问题给了身份、原因、字数这些信息"）
- 任务2：重点看学员是否在**同一个对话**中追问（不是开新对话），是否真的来回了3轮
- 任务3：重点看学员是否真的去搜了，而不是直接相信AI的回答

## 常见问题 FAQ

### Q1："AI工具要花钱吗？"

> "网页版有免费额度，日常使用够了。后面我们学到Claude Code的时候再讲付费的事情。现在不用担心费用问题。"

### Q2："AI会不会泄露我的隐私？"

> "这个问题问得好。原则是：**不要在AI对话中输入你的身份证号、银行卡号、密码这些敏感信息**。像刚才的简历练习，我让大家用化名，就是这个原因。AI是工具，跟在公共电脑上打字一样，不要输入你不愿意被别人看到的东西。"

### Q3："AI回答的内容我可以直接用吗？比如直接交给领导？"

> "可以用，但要先检查。特别是涉及数字、日期、法律条文、公司名称这些**事实性内容**，一定要验证。文字表达、格式这些可以直接用。养成一个习惯：用AI生成，用搜索验证，用自己的判断做最终决定。"

### Q4："AI说的东西跟搜索结果不一样，听谁的？"

> "听证据的。哪边有官方来源（政府网站、法律原文、官方公告），就信哪边。如果都没有明确官方来源，那就多找几个来源交叉对比。"

### Q5："我打字慢，跟AI对话效率很低怎么办？"

> "没关系，你可以先在记事本里把要问的话打好，然后复制粘贴到AI对话框里。而且AI不着急，你慢慢打它也会等你。另外，打字速度会越用越快，这本身就是在练。"

## 超时应急方案

如果时间不够，按以下优先级砍内容：

| 优先级 | 可砍内容 | 影响 | 处理方式 |
|:---:|:---|:---|:---|
| 先砍 | 任务1（对比练习） | 低——前面演示已经展示过对比 | 改为课后作业 |
| 再砍 | 角色设定演示 | 低——属于进阶技巧 | 口头提一句"AI还能扮演面试官"即可 |
| 最后砍 | 任务3（验证练习） | 中——但"信但验"的原则已在幻觉演示中传达 | 改为课后作业，下节课开头抽查 |

**绝对不能砍的部分：**
- 模糊 vs 精确提问对比演示（这是本课核心）
- AI幻觉演示（这是建立正确认知的关键）
- 任务2 多轮对话简历优化（这是最重要的实操）
