# 第8节：Claude Code与开发工作流

## 概要

> 本节是基础课的收官，也是从"学基础"到"做项目"的转折点。学员从网页版AI升级到Claude Code——一个能直接在终端里写代码、改文件、运行程序的AI开发搭档。本节要跑通一个完整的"需求→开发→运行"流程，为后续5节实践课打下方法论基础。

## 教案

### 课时信息

- 所属阶段：基础课（第8节/共8节）
- 主讲：沈林伟
- 辅助：梅杭俊
- 时长：45 分钟

### 教学目标

学完本节课，学员能够：
1. 在终端中启动Claude Code并与之对话
2. 用Claude Code完成"描述需求→生成代码→运行→调试"的完整流程
3. 理解Claude Code与网页版AI的区别（能直接操作文件和运行代码）
4. 理解项目文件结构的基本概念（代码文件、数据文件、输出文件）

### 内容要点

#### 一、从网页AI到Claude Code（10分钟）

- 回顾：前几节用网页版AI生成代码 → 手动复制 → 手动运行
- Claude Code的升级：它能直接在你的电脑上写文件、改代码、运行程序
- 演示对比：
  - 网页版：复制代码 → 粘贴到文件 → 终端运行 → 出错 → 回网页问 → 再复制修改（来回跳转）
  - Claude Code：直接描述需求 → 它帮你写好文件 → 直接运行 → 出错它直接帮你改（一条龙）
- 启动：终端输入 `claude`，进入对话

#### 二、完整开发工作流演示（15分钟）

- 用一个真实小任务演示全流程：
  - 需求："帮我写一个程序，读取当前文件夹里所有txt文件，统计每个文件的字数，生成一份汇总表"
  - Claude Code生成代码 → 自动创建文件 → 运行 → 看到结果
  - 追加需求："把汇总表按字数从多到少排序" → Claude Code修改代码 → 再运行
  - 出错处理：如果报错，让Claude Code自己看报错信息并修复
- 讲解项目结构：
  - 代码文件（.py）
  - 输入数据（.txt / .csv）
  - 输出结果
  - 一个项目 = 一个文件夹，里面放这些东西

#### 三、开发方法论总结（5分钟）

- 完整工作流：**描述需求 → AI写代码 → 读懂 → 运行 → 发现问题 → 描述问题让AI修 → 验证结果**
- "先定义什么算对"的习惯：开始之前先想清楚"成功是什么样的"
- L1/L2/L3的区别在这个流程中的体现：
  - L1：能走通这个流程
  - L2：能在"读懂"和"描述问题"环节做到有方向的调整
  - L3：能把这个流程完整地跑到部署

#### 四、练习（15分钟）

### 练习任务

| 序号 | 任务描述 | 类型 |
|:---:|:---|:---:|
| 1 | 启动Claude Code，让它帮你写一个程序完成一个自选的文件处理任务 | 个人 |
| 2 | 在Claude Code生成的代码基础上，追加一个新需求（如修改输出格式、增加筛选条件） | 个人 |
| 3 | 故意制造一个错误（如删掉一行代码），让Claude Code帮你修复 | 个人 |

### 教学提示

- 这是学员第一次用Claude Code，可能需要适应终端内对话的模式
- 重点演示"出错→让AI修"的循环，消除"写代码怕出错"的心理
- 项目结构的概念点到即止，后续实践课会反复强化
- 安全意识融入：提醒学员不要让AI执行自己不理解的危险操作（如删除文件），养成"看懂再确认"的习惯
