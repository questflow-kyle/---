### 投影2：网页版 vs Claude Code 对比（环节二用）

```
网页版AI的流程：
  你问AI → AI给代码 → 你复制 → 你切到终端 → 你粘贴保存 → 你运行
  → 出错？→ 回浏览器问AI → 再复制 → 再粘贴……

Claude Code的流程：
  你说需求 → 它直接写好文件 → 你运行
  → 出错？→ 直接告诉它 → 它直接改 → 你再运行

核心区别：Claude Code 能直接操作你电脑上的文件，不需要你当搬运工。
```

### 投影3：演示用需求和成功标准（环节三用）

```
  文件名          字数
  note1.txt       156

  note3.txt       97
```

```
一个项目 = 一个文件夹

```

### 投影5：完整开发工作流（环节五用）

```
完整开发工作流：

  ① 定义需求 → 想清楚"要做什么"和"做完了什么样"
  ② AI写代码 → 用Claude Code描述需求，让它帮你写
  ③ 读懂代码 → 大概看一下代码在干什么（不用逐行理解）
  ④ 运行验证 → 跑一下看结果对不对
  ⑤ 发现问题 → 结果不对？报错了？
  ⑥ 描述问题 → 把问题告诉Claude Code，让它改
  ⑦ 再次验证 → 改完了再跑，直到结果满意

你负责：①③④⑤（定义、读懂、运行、发现问题）
AI负责：②⑥（写代码、改代码）
```

### 投影6：L1/L2/L3 标准（环节五用）

```
L1 能用：能走通 ①→②→④ 这个流程
L2 能调：能做到 ③读懂 和 ⑤⑥有方向地调整
L3 能交付：能把这个流程跑到部署上线
```

## 五、备用方案

### 备用方案A：个别电脑 Claude Code 无法使用

**症状**：`claude` 命令报错、API key 无效、网络不通等

**应对**：
1. 先尝试快速修复（2分钟内能解决就修）：
   - API key 问题：重新配置 `export ANTHROPIC_API_KEY=...`
   - 版本过旧：`npm update -g @anthropic-ai/claude-code`
   - 网络问题：检查 WiFi 连接
2. 如果修不好，该学员与旁边同学共用一台电脑（两人一组，一人操作一人观察，轮流）
3. 检查点任务不受影响（用网页版AI即可）
4. 课后单独帮该学员修复环境

### 备用方案B：全部电脑 Claude Code 无法使用（网络中断或API服务故障）

**这是最坏情况，概率低但必须有预案。**

**应对**：
1. **检查点任务**：不受影响，本来就用网页版AI
2. **环节二（对比演示）**：改为纯讲解 + 截图展示，口头说明区别
3. **环节三（工作流演示）**：切换到网页版AI演示：
   - 在网页版AI中输入同样的需求
   - 手动将代码保存到文件：
     ```bash
     nano word_count.py
     ```
     （粘贴AI生成的代码，`Ctrl+O` 保存，`Ctrl+X` 退出）
   - 运行和验证流程不变
4. **环节四（练习）**：用网页版AI + 手动保存代替 Claude Code
5. 向学员解释："今天网络有点问题，Claude Code 暂时用不了。我们先用网页版AI走一遍流程，方法是一样的。下节课再用Claude Code。"

> **注意**：用网页版AI + 手动复制的方式体验会差一些（多了复制粘贴步骤），但核心教学目标（开发工作流方法论）不受影响。

### 备用方案C：检查点任务时间不够或大面积卡住

**应对**：
1. 如果3分钟后还没有人完成，缩短检查点时间，直接进入总结
2. 讲师在投影上快速走一遍完整流程（1分钟）
3. 把"大家卡住"本身当教学素材：

> "大家发现用网页版AI来回切换很麻烦对吧？这就是为什么我们要升级到Claude Code——它能帮你省掉这些中间步骤。"

### 备用方案D：课堂时间不够

**应对**：
- 参见教案中的"时间应急方案"
- 底线：确保学员至少看完 Claude Code 演示 + 了解开发工作流七步
- 练习可以缩短到只做任务1（3分钟）
- 方法论总结可以只投影七步流程，不展开讲
