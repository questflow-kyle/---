# 第11节：实践——网页应用开发

## 概要

> 本节是实践课的关键跨越——从命令行脚本到可视化的网页应用。学员第一次看到自己用AI做出的东西可以在浏览器里打开、能点击交互。这是"能做应用"认知的起点，也是区分脚本和应用的分水岭。

## 教案

### 课时信息

- 所属阶段：实践课（第3节/共5节）
- 主讲：沈林伟
- 辅助：尹伟
- 时长：45 分钟

### 教学目标

学完本节课，学员能够：
1. 理解网页的基本构成（HTML=内容、CSS=样式、JS=交互）
2. 用Claude Code生成一个可在浏览器中打开的网页应用
3. 修改网页的内容和样式
4. 理解"前端应用"与"脚本"的区别

### 任务说明

#### 核心任务：用AI开发一个个人工具网页

用Claude Code开发一个有实际用途的单页网页应用（如：个人记账器、单位换算工具、待办事项清单）。

#### 分层要求

| 等级 | 完成标准 |
|:---:|:---|
| L1 能用 | 用Claude Code生成网页，能在浏览器中打开并使用基本功能 |
| L2 能调 | L1 + 修改页面内容、颜色、布局；能说出HTML/CSS/JS各自负责什么 |
| L3 能交付 | L2 + 添加交互功能（如数据保存到本地存储、输入校验）；页面美观可用，可以发给别人使用 |

### 教学流程

| 时间 | 内容 |
|:---:|:---|
| 0-8分钟 | 讲师演示：用Claude Code 3分钟做一个计算器网页。讲解HTML/CSS/JS的角色分工 |
| 8-12分钟 | 学员选择自己要做的工具类型，用中文写出功能需求 |
| 12-35分钟 | 学员用Claude Code独立开发，讲师巡回指导 |
| 35-45分钟 | 展示环节：每人在浏览器中打开自己的作品，互相浏览 |

### 教学提示

- 这是学员第一次做"能看到界面"的东西，成就感会很强，充分利用这个动力
- HTML/CSS/JS的概念只需要最表层的理解，不要深入语法。关键是知道"改颜色改CSS、改文字改HTML、改功能改JS"
- 建议提供3-4个模板方向让学员选，避免需求太发散导致卡住
- 展示环节很重要——让学员互相看成果，形成正向激励
- 安全意识融入：提醒不要在网页中硬编码个人敏感信息

### 所需材料/设备

- [ ] 每人一台电脑（WSL + Claude Code + 浏览器）
- [ ] 投影仪
- [ ] 可选：3-4个网页应用方向的简要需求描述（打印，供学员参考选择）
